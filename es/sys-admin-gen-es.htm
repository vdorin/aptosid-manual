Activar servicios en aptosid

insserv (Pontos 2008-04 y versiones posteriores): Para iniciar/parar servicios ya instalados:

Lea cuidadosamente en /usr/share/doc/insserv/README.Debian las notas de la versión y las páginas man:

 $ man insserv
 $ man invoke-rc.d
 $ man update-rc.d
 google LSB headers

Para 'iniciar (start)':

/etc/init.d/<service> start

Para 'parar (stop)':

/etc/init.d/<service> stop

Para 'reiniciar (restart)':

/etc/init.d/<service> restart

Para evitar que un servicio arranque al inicio

update-rc.d <service> remove
[quitará los enlaces de arranque]

Para asegurarse de que un servicio arranque al iniciar el sistema [no siempre es requerido]

update-rc.d <service> defaults
[crear enlaces de arranque]

Para leer la lista de los servicios que están corriendo:

ls /etc/rc5.d

S significa que el servicio arrancará.
K significa que el servicio no arrancará.

Instalaciones Viejas (anteriores a Pontos 2008-04)

insserv en sistemas instalados antes de Pontos 2008-04

Advertencia: Las instalaciones viejas no tendrán insserv automáticamente debido a enlaces simbolicos o initscripts de usuarios. NO es recommendable instalar insserv en instalaciones anteriores a Pontos. Este paquete debe ser utilizado con cuidado, porque si las dependencias LSB son incorectas o ya hay initscritps, pueden dar como resultado un sistema que no arranque. Instalar insserv en una instalación ya existente puede necesitar cambios manuales.

Lea la documentación de Debian, y las notas de la versión antes de instalar y activar el insserv.

Gestor de Inicio: (Boot-Up Manager - bum) - Herramienta Gráfica de Configuración de Servicios

Si la lógica de la secuencia de arranque de un sistema Debian no es muy clara y familiar no debe jugar con los symlinks, permisos, etcétera. Para evitar desconfigurar el sistema, el Boot-Up Manager le ayudará a automatizar la configuración.

Boot-Up Manager es un Editor GUI de Configuración de Niveles de Ejecución que permite determinar qué servicios de init son llamados cuando se inicia o reinicia el sistema. Se muestra una lista de todos los servicios que pueden arrancar al inicio. Puede cambiar el estado de servicio, si o no, de servicios individuales.

apt-get install bum

Para utilizar el Boot-Up Manager GUI:

$ sux
password:
bum

Documentación detallada sobre el Boot-Up Manager (bum).

Matar un servicio o proceso

pkill es muy útil porque es legible por humanos y puede funcionar tanto como usuario normal como root en una terminal o en el ttyX.

pkill -n service

Si no esta seguro de cómo deletrear el proceso o servicio que desea matar: pkill <tab> <tab> da una lista

htop es una alternativa muy buena. (Intente: killall -1 primero; killall -9 es el último recurso).

Claves de root perdidas

No se puede recuperar una clave perdida, pero sí se puede establecer una nueva.

Primero arranque desde el LiveCD.

Como root, monte su partición-raíz (por ejemplo: /dev/sdb2)

mount /dev/sdb2 /media/sdb2

Ahora, haga chroot hacia su vieja partición-raíz y fije una nueva clave de acceso:

chroot /media/sdb2 passwd

Establecer nueva claves

Para cambiar su clave de usuario, como $ usuario haga:

$ passwd

Para cambiar su clave de superusuario, como # root haga:

passwd

Para cambiar una clave de usuario como administrador, como # root haga:

passwd <user>

Configuración correcta de dpi - Principios Básicos

Las configuraciones de dpi son problemáticas de adivinar, pero actualmente son calculadas perfectamente por X.

Resoluciones Correctas y Frecuencia de actualización

Cada monitor tiene una combinación perfecta para él, pero, desafortunamente, no todos reportan los valores DCC correctos y a veces es necesario hacer un control manual.

Preselecciones de Fuentes de letras, Renderización y Tamaños

aptosid utiliza fuentes preseleccionadas (por Debian) que se han comprobado como muy equilibradas; si hace cambios en la selección de fuentes podría deteriorar la calidad de la renderización. Pero hay unas opciones poderosas en Debian (aparte de las de KDE>systemsettings) que puede proporcionar una renderización perfecta también con otras fuentes. Sea consciente de que cada fuente tiene solamente unos pocos tamaños perfectos, y que otros tamaños pueden ocasionar conflictos.

Variar el tamaño del "dpi" con este comando puede ser de ayuda:

fix-dpi-kdm

Debe mostrar el DPI para el tamaño del pantalla, pero usted también puede jugar con esto. Tendrá que cambiar a "init 3" y después a "init 5" para que funcione, o volver a arrancar.

Después de haber cambiado el tipo de fuente de letra o el DPI (en la configuración de X o en Firefox/Iceweasel), puede ser que necesite algunos ajustes para conseguir unos resultados que le gusten, especialmente después de un cambio de tipo Bitmap a uno de TrueType o viceversa:

dpkg-reconfigure fontconfig-config

Escoja nativo y autohinter en modo automático. Si no, varíelo a ver los resultados.

Si eso no funciona, puede que tenga que rehacer el cache de fuentes de letras con lo siguiente:

apt-get install --reinstall --yes -o DPkg::Options::=--force-confmiss -o DPkg::Options::=--force-confnew fontconfig fontconfig-config
Aplicaciones basadas en GTK como el Firefox/Iceweasel

Las aplicaciones basadas en GTK, son problemáticas en general con las fuentes de letras preseleccionadas por Kde. Esto se puede resolver con los siguientes pasos:

apt-get install gtk2-engines system-config-gtk-kde gtk-qt-engine gtk2-engines-qtcurve

En System Settings >Appearance tendrá una nueva entrada de menú llamada GTK Styles and Fonts. Establezca que 'GTK Styles' use 'Clearlooks' y haga que 'GTK Fonts' utilice 'KDE fonts' o bien experimente con las varias opciones hasta cumplir con sus preferencias.

CUPS

KDE tiene una sección incorporada en su Ayuda; sin embargo, los dist-upgrades a veces pueden causar algunos fallos en el CUPS. Aquí está una solución conocida y probada:

modprobe lp
echo lp >> /etc/modules
apt-get remove --purge cupsys cups
apt-get install cups

apt-get install cups cups-driver-gutenprint hplip

Asegúrese de que CUPS se esta ejecutando:

/etc/init.d/cups restart

A continuación en el navegador (browser) de internet:

http://localhost:631

Otro truco, es cuando al configurar CUPS por el GUI aparece una casilla pidiendo la clave de root pero la casilla ya contiene su nombre de usuario, entonces cuando entra su clave de usuario, no funciona!. Lo que realmente está pidiendo es que cambie su nombre de usuario a root y entre su clave de root.

El sonido en aptosid

El sonido está anulado por defecto en aptosid.

La versión KDE usa Kmix y la XFCE usa Mixer.

La mayor parte de las veces se trata solo de hacer click sobre el icono de sonido en la barra de tareas y quitar la señal en 'Mute box'.

Kmix

En Kmix necesitará activar las opciones preferidas para el canal de opciones, Kmix>Setting>Configure Channels.

XFCE

En XFCE haga correr la aplicación del mixer y añada algunos controles mediante Multimedia>Mixer y haga click en la caja Select Controls. O bien en un terminal:

$ xfce4-mixer

Alsamixer

Si prefiere usar Alsamixer, está en el paquete alsa-utils:

apt-get update
apt-get install alsa-utils
exit

Establezca su configuración preferida de sonido como $user en el terminal:

$ alsamixer
