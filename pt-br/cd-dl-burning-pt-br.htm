Como baixar o ISO do aptosid

OBSERVAÇÃO MUITO IMPORTANTE: o aptosid, como LiveCD do Linux, é baseado em tecnologia de alta compressão. Porisso, cuidados especiais são necessários ao queimar a imagem ISO: use apenas mídias de alta qualidade [CD ou DVD+RW] e queime no modo DAO (disk-at-once) e em velocidade não superior a 8x.

SEMPRE FAÇA BACKUPS (CÓPIAS) DE SEUS DADOS!

Definições dos arquivos em cada espelho

Cada espelho do aptosid apresenta os seguintes arquivos:

MD5SUM
MD5SUM.gpg
SHA256SUM
SHA256SUM.gpg
SOURCES
aptosid-20xx-xx-release-name-window-manager-arch-datetimestamp.arch.manifest
aptosid-20xx-xx-release-name-window-manager-arch-datetimestamp.iso

O arquivo .manifest lista todos os pacotes contidos na iso.

O arquivo com extensão .iso é a imagem a ser baixada.

Os arquivos .md5 e .sha256 são usados na verificação da integridade da ISO.

Os arquivos .gpg são arquivos de assinatura, de forma que você possa verificar se a cadeia de caracteres ('checksums') (.md5 .sha256) não foi alterada e pode ser usada para confirmar a integridade da ISO.

Os fontes interessam apenas àqueles que precisarem de uma cópia estática do código-fonte para fins de redistribuição e ter acesso aos requisitos da licença. Leia o arquivo SOURCES para mais informações.

Muito obrigado á webtropia.com que patrocina aptosid e a todos os espelhos ("mirrors") pelo seu esforço ao oferecer nos seus servidores o aptosid.

Conferência do md5sum e validação

OBSERVAÇÃO MUITO IMPORTANTE: o aptosid, como LiveCD do Linux, é baseado em tecnologia de alta compressão. Porisso, cuidados especiais são necessários ao queimar a imagem ISO: use apenas mídias de alta qualidade [CD ou DVD+RW] e queime no modo DAO (disk-at-once) e em velocidade não superior a 8x.

Esta conferência é usada para checar a integridade de arquivos. A soma do md5sum encontrada é comparada com uma soma já conhecida. Assim, você pode verificar se um arquivo foi alterado ou sofreu danos; essa checagem é altamente recomendada para arquivos baixados pela internet.

Se o valor do md5 do arquivo baixado corresponder ao valor do md5 do arquivo original, você pode estar seguro de que o arquivo foi baixado corretamente. No Linux, a verificação é feita com este comando (o cálculo demora um pouquinho):

$ md5sum arquivo_a_verificar

A soma será escrita no próprio terminal.
Esta validação também pode ser feita no Windows. Basta instalar o programa "md5sum" (486 kb).

As imagens ISO do aptosid são sempre acompanhadas pelo arquivo de checagem do md5 (ele possui a extensão .md5) e você deve sempre validá-las antes de queimá-las. Isto garante que, se ocorrerem problemas, eles não são causados pelo arquivo que você baixou e ainda mantém o fórum do aptosid livre de questões não solucionadas só porque a ISO estava danificada. Isto é feito no terminal. Mude para o diretório onde estão os arquivos com a imagem ISO e com o arquivo .md5 e digite o seguinte comando para validar a soma md5sum:

Baixe os seguintes arquivos, diretamente do espelho:

MD5SUM
MD5SUM.gpg
SHA256SUM
SHA256SUM.gpg

Agora, digite no terminal:

$ md5sum -c MD5SUM

Se os valores encontrados não forem os mesmos, você verá uma mensagem de erro:

aptosid-2011-03-ponos-kde-full-i386-amd64-201112312100.iso: OK
aptosid-2011-03-ponos-kde-lite-amd64-201112312100.iso: OK
aptosid-2011-03-ponos-kde-lite-i386-201112312100.iso: OK
aptosid-2011-03-ponos-xfce-amd64-201112312100.iso: OK
md5sum: aptosid-2011-03-ponos-source.tar: No such file or directory
aptosid-2011-03-ponos-source.tar: FAILED open or read
aptosid-2011-03-ponos-xfce-i386-201112312100.iso: FAILED
md5sum: WARNING: 1 listed file could not be read
md5sum: WARNING: 1 computed checksum did NOT match

Se o arquivo tiver sido baixado corretamente, é listado com OK .

Se você quiser verificar a integridade dos arquivos .gpg, faça o seguinte como $USUÁRIO:

Exemplo usando:
aptosid-2011-02-imera-kde-lite-i386-201107131633.iso

1) Importe a chave pública de "Stefan Lippers-Hollmann's" (slh). (An alternate keyserver is subkeys.pgp.net ):

$ gpg --keyserver wwwkeys.pgp.net --recv-keys E3BD538B
gpg --edit-key E3BD538B
Command> trust
Command> (choose the option you prefer or quit) ('escolha uma opção ou saia')
Command> quit

2) Agora você pode confirmar a integridade do 'checksum' com:

$ gpg --verify SHA256SUM.gpg SHA256SUM

Se você classificou a chave como totalmente confiável, usando os comandos correspondentes do gpg, então esta é a saída que você verá (se tiver escolhido a opção 5 - I trust ultimately, como sugerimos):

$gpg: Signature made Wed 15 Jul 2009 08:20:59 EST using DSA key ID E3BD538B
$ gpg: Good signature from "Stefan Lippers-Hollmann (aptosid.com) <s.l-h@gmx.de>"

Você vai ver um 'Good signature' (Assinatura boa) acompanhado de um AVISO, se você não tiver classificado a chave:

gpg: Signature made Wed 15 Jul 2009 08:20:59 EST using DSA key ID E3BD538B

gpg: 3 marginal(s) needed, 1 complete(s) needed, PGP trust model
gpg: depth: 0  valid:   2  signed:   0  trust: 0-, 0q, 0n, 0m, 0f, 2u
gpg: Good signature from "Stefan Lippers-Hollmann (aptosid.com) <s.l-h@gmx.de>"

Como eu queimo o CD no Windows?

OBSERVAÇÃO MUITO IMPORTANTE: o aptosid, como LiveCD do Linux, é baseado em tecnologia de alta compressão. Porisso, cuidados especiais são necessários ao queimar a imagem ISO: use apenas mídias de alta qualidade [CD ou DVD+RW] e queime no modo DAO (disk-at-once) e em velocidade não superior a 8x.

Naturalmente, é possível queimar o CD no Windows, também. O arquivo baixado deve ser queimado como um arquivo ISO. Se o Winrar (ou qualquer outro programa de compressão) estiver associado a arquivos .ISO, podem ocorrer problemas, porque ele será visto como um simples arquivo. Portanto, de posse da imagem ISO, queime logo um CD. Não queime o arquivo ISO como disco de dados!! É necessário queimá-lo como Disk-At-Once

Existem vários e bons programas que você pode usar no Windows:

Queimadores de código aberto

cdrtfe : compatível com Windows 9x/ME/2000/XP (testado no Win95, Win98SE, Win2000 e WinXP); somente no Win9x/ME: Camada ASPI funcional (p.ex. Adaptec ASPI 4.60)

LinuxLive USB Creator, um projeto open source project, com uma GUI para MS Windows™, wue torne possível instalar aptosid-i386.iso (32 bit) num dispositivo USB. Antes de arrancar(boot) vai ter que utilizar o cheatcode fromhd na linha de entrada do grub.

Queimadores de código fechado e proprietários
  • Nero (não o Nero Express!)
  • Outra boa ferramenta freeware é CD/DVD Burner XP pro
  • Burncdcc da terabyteunlimited (só queima ISOs...)

Como queimar o CD no Linux

OBSERVAÇÃO MUITO IMPORTANTE: o aptosid, como LiveCD do Linux, é baseado em tecnologia de alta compressão. Porisso, cuidados especiais são necessários ao queimar a imagem ISO: use apenas mídias de alta qualidade [CD ou DVD+RW] e queime no modo DAO (disk-at-once) e em velocidade não superior a 8x.

Se você já roda Linux em sua máquina, você pode queimar o CD tanto pela linha de comando quanto usando algum programa já existente no sistema operacional.

Para tarefas desse tipo, o aptosid e outras distribuições instalam o K3b por padrão. Basta abri-lo, clicar em Ferramentas -> Queimar imagem de CD e selecionar o arquivo ISO a ser queimado (p. ex. aptosid.iso).

O K3B inicialmente calcula a soma do MD5 para a imagem ISO (o que demora um momento). Se o resultado for igual ao apresentado pelo arquivo MD5 (isto é, o aptosid.iso.md5), você pode queimar a imagem, bastando para tal clicar em "Queimar".

A maioria dos problemas na hora de queimar acontecem com o 'frontend'. Você pode queimar diretamente pelo terminal com burniso.
