Definitions of files on each mirror

Each sidux mirror has following files:

MD5SUM
MD5SUM.gpg
SHA256SUM
SHA256SUM.gpg
SOURCES
sidux-20xx-xx-release-name-window-manager-arch-datetimestamp.arch.manifest
sidux-20xx-xx-release-name-window-manager-arch-datetimestamp.iso

The .manifest file lists all the packages contained in the specific iso.

The .iso is the image to download.

The .md5 and .sha256 files are to check the integrity of the iso.

The .gpg files are signature files, so you can verify that the checksums (.md5 .sha256) have not been changed and that they can be used to verify the integrity of the iso.

The sources tar is only for those who require a static copy of the source code for redistribution purposes to comply with the licensing requirements. Read the SOURCES file for more information.

md5 сума и верификация

ИЗКЛЮЧИТЕЛНО ВАЖНО: sidux като Linux LIVE-CD е базиран на технология с висока компресия на данните на диска, поради което е нужно специално внимание при записа на ISO-образа върху CD/DVD/CD/DVD-RW диск. Използвайте само висококачествени дискове, и записвайте в DAO-режим (disk-at-once), и със скорост не по-висока от x8.

Това е контролна сума за сваления файл, и се използва да провери целостта му. Изчислената md5 сума на файла се сравнява с предварително известната. По такъв начин вие валидирате дали файла не е бил изменен или повреден; това се препоръчва за файлове свалени от Интернет.

Ако md5 сумата на сваления файл е същата като тази в md5-файла, можете да сте сигурни че файлът се е свалил правилно. Под Linux можете да получите контролната сума на сваления файл както следва: (изчислението отнема няколко секунди)

$ md5sum файл_за_проверка

Сумата ще бъде изписана в конзолата.
С md5sum (486 kb) md5 сумата може също да бъде изчислена в Windows.

ISO-образите на sidux винаги се съпътстват от md5sum файл, и трябва да бъдат проверявани преди записване на диск. Това се прави в конзолата. Преминете в директорията където се намират ISO-файла и файла на MD5 сумата, и изпълнете тази команда за да проверите:

Download the following files from the mirror:

MD5SUM
MD5SUM.gpg
SHA256SUM
SHA256SUM.gpg

In a terminal:

$ md5sum sidux.iso

Ако контролните суми не съвпадат, ще получите съобщение за грешка:

"sidux.iso: Error
md5sum: Warning: calculated checksum does not match!"

Ако сваленият файл е коректен, проверката не издава никакво съобщение.

If you wish to be able to verify the validity of the .gpg files do the following as $USER:
Example using:
aptosid-2011-02-imera-kde-lite-i386-201107131633.iso

1) Import "Stefan Lippers-Hollmann's" (slh's) public key from a key-server. (An alternate keyserver is subkeys.pgp.net ):

$ gpg --keyserver wwwkeys.pgp.net --recv-keys E3BD538B
gpg --edit-key E3BD538B
Command> trust
Command> (choose the option you prefer or quit)
Command> quit

2) Now you can verify the integrity of the checksum with:

$ gpg --verify SHA256SUM.gpg SHA256SUM

If you have classified the key as trustworthy using the necessary gpg commands, then this is the output you should get with "5 trusted ultimately":

$gpg: Signature made Wed 15 Jul 2009 08:20:59 EST using DSA key ID E3BD538B
$ gpg: Good signature from "Stefan Lippers-Hollmann (sidux.com) <s.l-h@gmx.de>"

You will get 'Good signature' with a WARNING if you did not choose to classify the key:

gpg: Signature made Wed 15 Jul 2009 08:20:59 EST using DSA key ID E3BD538B
gpg: checking the trustdb
gpg: 3 marginal(s) needed, 1 complete(s) needed, PGP trust model
gpg: depth: 0  valid:   2  signed:   0  trust: 0-, 0q, 0n, 0m, 0f, 2u
gpg: Good signature from "Stefan Lippers-Hollmann (sidux.com) <s.l-h@gmx.de>"

Как за запишем CD под Windows?

ИЗКЛЮЧИТЕЛНО ВАЖНО: sidux като Linux LIVE-CD е базиран на технология с висока компресия на данните на диска, поради което е нужно специално внимание при записа на ISO-образа върху CD/DVD/CD/DVD-RW диск. Използвайте само висококачествени дискове, и записвайте в DAO-режим (disk-at-once), и със скорост не по-висока от x8.

Естествено, CD-то може да бъде записано и под Windows също. Сваленият файл трябва да бъде записван като ISO-файл. Възможно е Winrar (или друга подобна архивна програма) да го отваря като архив; но това е неправилно тъй като не е нужно да се отваря като архив, а да се запише върху CD диска "наведнъж", като ISO-образ.

Има няколко възможности за запис под Windows.

Свободен софтуер за записване на CD под Windows

cdrtfe :съвместим с Windows 9x/ME/2000/XP (тестван с Win95, Win98SE, Win2000, and WinXP), само за Win9x/ME: нужен е работещ ASPI Layer (т.е. Adaptec ASPI 4.60)

LinuxLive USB Creator, an open source project, offers a GUI for MS Windows™, which enables an install of a sidux-i386.iso (32 bit) to a USB stick. You need to use fromhd bootline/cheatcode on the boot line.

Несвободни и проприетарни записвачки под Windows
  • Nero (не nero express!)
  • Друга безплатна възможност е CD/DVD Burner XP pro
  • BurnCDCC от terabyteunlimited - може да записва само ISO-образи.

Записване на CD под Линукс

ИЗКЛЮЧИТЕЛНО ВАЖНО: sidux като Linux LIVE-CD е базиран на технология с висока компресия на данните на диска, поради което е нужно специално внимание при записа на ISO-образа върху CD/DVD/CD/DVD-RW диск. Използвайте само висококачествени дискове, и записвайте в DAO-режим (disk-at-once), и със скорост не по-висока от x8.

Ако вече имате Линукс на вашия компютър запишете CD-диска или от конзолата, или с която и да е друга от инсталираните записващи програми.

В sidux и други Линукс дистрибуции, K3b е една от популярните CD-записващи програми. Отворете я, изберете "Tools" -> "burn CD-Image..." и укажете ISO-файла за записване (т.е. sidux.iso).

K3B в началото пресмята контролната MD5 сума на ISO-файла. Когато тя съвпада с контролната сума на MD5-файла (в случая sidux.iso.md5) то свалянето е било успешно, и можете да го запишете на диска натискайки бутона "Start".

Problems with burning mostly happen with a frontend. You can burn directly from the shell with burniso.

Also see Writing sidux to a USB/SSD stick with any Linux, MS Windows or Mac OS X system (32 and 64 bit).
