Sources list e driver firmware: non-free

Sources list

Una iso di aptosid fornisce soltanto software conforme alle dfsg (Debian Free Software Guidelines). Probabilmente si vorranno aggiungere le sezioni contrib non-free al file /etc/apt/sources.list.d/debian.list :

mcedit /etc/apt/sources.list.d/debian.list

Adesso le linee dovrebbero assomigliare a queste, a seconda della scelta del mirror, (fare riferimento a Lista di server Debian e mirror e del loro stato attuale):

# Debian
deb http://ftp.de.debian.org/debian unstable main contrib non-free
# deb-src http://ftp.de.debian.org/debian unstable main contrib non-free

Dopo aver modificato le sorgenti software si deve eseguire un apt-get update prima di effettuare ricerche o installazioni di nuovi pacchetti (eseguire apt-get update è una buona pratica prima dell'installazione di qualsiasi pacchetto).

Rilevazione del firmware - non-free

Comandi per installare eventuali firmware non-free (si presume che le voci contrib non-free siano state già aggiunte alle fonti del software):

apt-get update
apt-get install firmware-linux firmware-linux-free firmware-linux-nonfree

Il firmware richiesto può essere determinato con la seguente enumerazione device/firmware oppure usando lo script fw-detect (contenuto nel pacchetto aptosid-scripts).

$ fw-detect

La risposta di fw-detect descrive i comandi necessari per installare e attivare il firmware:

Esempio:

#Detected driver that requires firmware to operate
#Follow these instructions to obtain the correct firmware
# and activate the zd1211rw driver:
apt-get update
apt-get install zd1211-firmware
modprobe -r zd1211rw
modprobe zd1211rw

Per installare firmware dai repository git eseguire:

apt-get install git-core
firmware non-free deb su di una penna usb

Nel caso si abbia bisogno di firmware in formato .deb, metterli in una penna usb per trasferirli su di un altro computer, si possono scaricare sia come zip che come tar.gz da http://cdimage.debian.org/cdimage/unofficial/non-free/firmware/sid/current/ ed estrarli in una directory di nome firmware, se ne dovrebbero vedere una certa quantità.

Quindi scaricare firmware-linux-free*.deb ed aggiungerlo agli altri .deb nella directory in cui si sono estratti i firmware nonfree. Quindi trasferirli con la penna usb nel computer che necessita di firmware non-free ed eseguire:

dpkg -i firmware-linux-nonfree*.deb

Cercheremo di fornire i pacchetti per i firmware legalmente distribuibili sui nostri archivi software "non-free" ma non tutti i produttori lo permettono.
