Назначение файлов на "зеркалах"

Следующие файлы содержатся на каждом aptosid сервере:

MD5SUM
MD5SUM.gpg
SHA256SUM
SHA256SUM.gpg
SOURCES
aptosid-20xx-xx-release-name-window-manager-arch-datetimestamp.arch.manifest
aptosid-20xx-xx-release-name-window-manager-arch-datetimestamp.iso

Файл .manifest содержит все пакеты соответствующего ISO-образа.

ISO-образ является предлагаемым для скачивания файлом.

Файлы .md5 and .sha256 предназначены на проверку целостности ISO-образа.

Файлы .gpg являются сигнатурами, с помощью которых можно проверить файлы (.md5 .sha256) не были ли они изменены. Последние используются для проверки целостности ISO-образа.

Архив источников aptosid*-source.tar предназначен только для тех, которые нуждаются в статической копии первоначального кода для передач, чтобы наполнять требования лицензирования.

Сумма md5sum и проверка целостности загруженных файлов

Важная информация: aptosid на Linux-LIVE-DVD/CD сильно сжат. По этой причине нужно обратить особое внимание на запись ISO. Мы рекомендуем для записи высококачественные CD (или DVD+RW), запись в режиме DAO (disk-at-once) и не быстрее чем 8x.

Сумма md5sum - это контрольная сумма файла, используемая для проверки целостности соответствующего файла. При этом вновь полученная сумма md5sum файла сравнивается с известной предыдущей суммой. Таким образом, можно установить, был ли изменен файл или поврежден, что целесообразно при загрузке файлов из интернета.

Если суммы md5sum и загруженного файла совпадают, то файл скачан без ошибок. В Linux сумму MD5 файла можно получить так:

$ md5sum имя_файла (который вы хотите проверить)

Затем сумма выводится на консоль. С помощью md5sum (486 Кб) MD5-сумма может быть рассчитана и под Windows.

ISO-образы aptosid всегда предлагаются для загрузки с соответствующей суммой md5sum и всегда должны проверяться перед записью на CD. Это должно выполняться в директории содержащей файл ISO и сумму MD5 следующим образом:

Скачайте следующие файлы с зеркала:

MD5SUM
MD5SUM.gpg
SHA256SUM
SHA256SUM.gpg

В терминале:

$ md5sum aptosid.iso

Проверяют сумму md5sum. Если обе проверочные суммы не совпадают, будет получено сообщение об ошибке:

aptosid-имя.iso: неудача md5sum: предупреждение: 1 от 1 рассчитанной проверочной суммы не совпадают.

Если файл был скачан без ошибок, суммы должны совпасть и запись закончится без какого-либо сообщения.

Если Вы хотите проверить соответствие файлов .gpg, сделайте следующее с правами пользователя:

Пример:
aptosid-2011-02-imera-kde-lite-i386-201107131633.iso

1) Импорт общедоступного ключа "Stefan Lippers-Hollmann's" (slh's) с сервера. (An alternate keyserver is subkeys.pgp.net ):

$ gpg --keyserver wwwkeys.pgp.net --recv-keys E3BD538B
gpg --edit-key E3BD538B
Command> trust
Command> (choose the option you prefer or quit)
Command> quit

2) Подтверждение действительности проверочных сумм:

$ gpg --verify SHA256SUM.gpg SHA256SUM

В случае, если этот ключ с помощью команд GPG указан как верный, Вы получите сообщение: "5 trusted ultimately".

$gpg: Signature made Wed 15 Jul 2009 08:20:59 EST using DSA key ID E3BD538B
$ gpg: Good signature from "Stefan Lippers-Hollmann (aptosid.com) <s.l-h@gmx.de>"

Сигнал 'Good signature' с ПРЕДУПРЕЖДЕНИЕМ приходит, если ключ не был проверен.

Прожиг Live-CD в Windows

Важная информация: aptosid на Linux-LIVE-DVD/CD сильно сжат. По этой причине нужно обратить особое внимание на запись ISO. Мы рекомендуем для записи высококачественные CD (или DVD+RW), запись в режиме DAO (disk-at-once) и не быстрее чем 8x.

Само собой разумеется, CD можно прожигать и в Windows. Загруженный файл должен прожигаться как ISO-файл. Winrar (или другая программа архивирования), связанный с ISO по умолчанию, открывает ISO-образ как простой архивный файл. Но ISO-образ используется для записи CD.

Имеется ряд программ для записи ISO в Windows.

LinuxLive USB Creator - проекта с открытым кодом - предлагает графический интерфейс для MS Windows™, который позволяет установку aptosid-i386.iso (32 бит), придерживаться USB. Вы должны использовать чит-код fromhd на командной линии grub.

Запись ISO-образа под Linux

Важная информация: aptosid на Linux-LIVE-DVD/CD сильно сжат. По этой причине нужно обратить особое внимание на запись ISO. Мы рекомендуем для записи высококачественные CD (или DVD+RW), запись в режиме DAO (disk-at-once) и не быстрее чем 8x.

На компьютере с уже установленным Linux можно записать LiveCD любой установленной в системе программой для записи дисков. В aptosid - k3b - стандартная программа для записи дисков. В пункте меню необходимо выбрать "Tools" -> ...Записать образCD (или DVD)" и выбрать записываемый ISO-образ (например, aptosid-Имя.iso). Установите режим DAO (Disk At Once) как метод для записи дисков.

K3b сначала рассчитывает сумму MD5 ISO-образа (занимает некоторое время). Совпадение рассчитанного значения MD5 со значением проверочной суммы MD5 файла, сохраненного при загрузке образа и находящегося в той же самой папке (например, aptosid-Имя.iso.md5) означает, что загрузка прошла успешно и файл может быть записан простым щелчком на кнопке "Начать".

Нажав правой кнопкой мыши на строке, содержащей рассчитанную сумму, выберите пункт "compare checksums", это вызовет дополнительное окно, в которое можно вручную ввести известную MD5 сумму для сравнения.

Проблемы с прожигом главным образом случаются из-за внешнего интерфейса. Вы можете прожигать непосредственно из консоли с помощью burniso.
