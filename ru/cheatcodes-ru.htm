Чит-коды

Синтаксис использования данных из поля "значение" таблицы: чит-код=значение. Если, например, желаемое разрешение экрана - 1280x1024, то в командную строку grub необходимо ввести screen=1280x1024, для использования русского языка lang=ru.

Ссылка на полный список с загрузочными чит-кодами

Особые чит-коды для ISO-образа

Чит-коды Значение Описание
blacklist имя модуля временно отключить модули до запуска udev
desktop kde выбор менеджера окон: kde
fluxbox выбор менеджера окон: fluxbox
fromiso смотрите в главе "fromiso"
hostname myhostname изменяет имя хоста системы на ISO-образа
lang be, bg, cz, da, de, de_CH, el, en, en_AU, en_GB, en_IE, es, fr, fr_BE, ga, hr, hu, it, ja, nl, nl_BE, pl, pt (pt_BR), pt_PT, ro, ru, zh устанавливает Ваш язык, и также основную локализацию (locale), расположение символов на клавиатуре (в консоле и в X-окружении), временной пояс и ближайшее зеркало репозитория Debian.
Можно использовать также синтаксис lang=ll_cc или lang=ll-cc. Таким образом, ll выбирает язык, а cc клавиатуру, зеркало репозитория и временной пояс (e.g. lang=fr-be). По умолчанию для английского языка установлено en_US с с временным поясом UTC и для немецкого языка de_DE с временным поясом Europe/Berlin.
Например: lang=pt_US tz=Pacific/Auckland
md5sum проверяет, в порядке ли загруженный CD
noaptlang отключает инсталляцию пакетов локализации, основанных на выбранном языке
nodhcp не использовать DHCP (DHCP пробует автоматически создать Ethernet соединения)
noeject не удаляет CD из дисковода
nofstab не пишет новый fstab
nointro пропускает вывод index.html при запуске ISO-образа
nocpufreq дезактивирует Speedstep/Powernow
nonetwork дезактивирует автоматическую конфигурацию сетевых адаптеров при загрузке
noswap дезактивирует раздел swap
persist смотрите в главе "fromiso и persist"
smouse ищет посредством hwinfo серийные устройства ввода мыши
toram копирует CD в RAM и стартует с копии в RAM
tz tz=Europe/Dublin устанавливает Ваш часовой пояс. Если Ваш BIOS-часы уставлены на UTC, используете utc=yes. После копирования и вставки в адресную строку браузера: file:///usr/share/zoneinfo/ Вы наидёте список всех поддержанных часовых поясов.
Чит-коды для графического X-сервера

Дополнительно должны использоваться также либо чит-код xrandr, либо чит-код xmodule, если применяют чит-коды для сервера X для графических карт Radeon, Intel или MGA.

Чит-коды Значение Описание
dpi auto или XX установите желательные точки на дюйм для монитора. Вы можете вычислить точки на дюйм экрана от ширины в пикселях, разделенных на диагональный размер в дюймах, и умножать это на 1.25 для 4:3-экран, на 1.18 для 16:10-экран или на 1.147 для 16:9-экран. Для 24"-монитора с 1920x1080-экраном это было бы 1.147*1920/24, дающий dpi=92 или для 15"-монитора с 1600x1200ßэкраном было бы 1.25*1600/15, дающим dpi=133.
hsync 80 (например) устанавливает горизонтальную X-частоту развертки монитора в килогерцах
noml вынуждает конфигурацию X.org обходиться без "modelines" для использования RandR 1.2 драйверов.
noxrandr дезактивирует применение расширений RandR 1.2 новыми драйверами X.org и использует старые техники для опроса свойств монитора
screen 1280x1024 устанавливает определенное пользователем разрешение для X (1260x1024 или другие разрешения дисплея)
vsync 60 (например) устанавливает вертикальную синхронизацию X
xdepth значения: 8 15 16 24 устанавливает глубину цвета, которую использует X.org (не все драйверы поддерживают 1 и 4)
keytable (обычно us, de) устанавливает расположение клавиш на клавиатуре, которое используется X.org
xkbmodel (обычно) pc105 устанавливает тип клавиатуры, который используется X.org (число обозначает количество клавиш)
xkboptions (e.g.) grp:alt_shift_toggle выбор варианта раскладки используясь X.org
xkbvariant (e.g.) nodeadkeys, устанавливает вариант раскладки
xmode (обычно 800x600) устанавливает размер экрана по указанным данным, 1024x768, 1600x1200 etc
xmodule or xdriver ati, fbdev, i810, intel, mga, nv, radeon, savage, vesa использует выбранный модуль X
xrandr вынуждает конфигурацию X.org использовать расширения RandR 1.2
xrate XX вынуждает установку предпочитаемой частоты при драйверах, которые поддержаны RandR 1.2. Эта опция должна использоваться в сочетании с загрузочной опцией xmode. Подробную документацию находите здесь.
xhrefresh 75 (например) устанавливает горизонтальную частоту в X (монитора) в килогерцах
xvrefresh 60 (например) устанавливает вертикальную частоту  в X (монитора) в герцах
