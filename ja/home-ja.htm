/home のバックアップ

/homeを移動する前にまず、 /home の下にあるファイルすべてを(rootとして)圧縮ファイルにバックアップします: 

tar cvzpf  somewhere/home.tar.gz /home

このファイルを解凍するには:

tar xvpf somewhere/home.tar.gz

rdiffを使ってバックアップする別の方法

/home の移動

Do not use or share an existing $home from another distribution as the $home configuration files in a home directory will conflict if you share the same username between differing distributions.

Moving or using an existing aptosid /home can be done two ways, one with livecd, the other with command line, neither way is difficult.

パーティションの UUID情報が起動時のマウントの際必要なため、新しいパーティションのUUIDが必要です。

一番簡単な方法は、実際に /home を移動する前に新しいUUIDを /etc/fstab にコメント行として書き込んで置くことです。.

PC を起動してGRUBメニューの起動オプションの行に 1 とタイプしてエンターキーを押します。これで init (rannrevel) 1、つまりシングルユーザーモードでスターとするため、あなたのホームディレクトリは使われていないので、安全に作業することができます。

ttyにrootとしてログインし、新しく /home にしたいパーティションを別の名前でマウントします, 例として:

mount /dev/sdxX /media/new-home
(sdxXを実際のパーティションのデバイスに交換してください)
cp -pr /home /media/new-home

次に新しく /home になるパーティションに合うように /etc/fstab を編集します: 

mcedit /etc/fstab

ここで先ほど用意した新しい /home のコメント行から最初の# を消して有効行にして、一方古い /home の行頭には # を加えて コメント行にします。 そしてF2でファイルを保存し、F10で終了し、再起動します。
