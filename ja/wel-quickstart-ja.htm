aptosidクイックスタートガイド

重要事項！: aptosidはLinuxのLIVE-CDのひとつとして高圧縮技術に基づいています。そのためISOイメージをCDに焼くときには特別の配慮が必要です。高品質のCD[またはDVD+RW]を選び、必ずDAO-mode (disk-at-once)を使ってx8以下のスピードで焼いてください。

aptosidはDebian sidと100%コンパチブルであることを目指していますが、ときに一時的なホットフィックスを使ってDebianのパッケージの代わりのパッケージを提供することがあります。 aptosidのレポジトリーにはアート、カーネル、スクリプト、ツール、文書などのaptosid独自のパッケージが置いてあります。

まずこれだけは読んでほしいページ

Linux は初めて、または aptosid は初めてという人にまずこれだけは読んでほしいというページがあります。このクイックスタートのページもその一つですが、他に:

  • ターミナル/コンソール - テキストコンソールの使い方、sux コマンドの使い方を説明します。

  • Non-free ドライバとソースについて - Non-free のドライバの入手とインストールについて述べています。

Debian sid/unstableの安定性

sidはDebianのunstableレポジトリの名前です。Debian sidは頻繁にアップデートされアップストリームの最新のソフトウェアバージョンとシンクロナイズされます。このアップデートの頻繁さのために、アップストリームからDebianのパッケージになるまでの短い期間にパッケージをとことんテストすることはできません。

aptosidカーネル

aptosidではDebianのカーネルを使っていません。aptosidのカーネルは http://www.kernel.org/ の最新のカーネルに、様々なハードウェアに対応し、問題点を避け、スピードと安定性を増すように変更を加えaptosidのために最適化されたカーネルです。

カーネルのミラーは: カーネルのアップグレードのページにあります。

パッケージ管理

aptosidはDebianのパッケージシステムに準拠して /etc/sources.list.d/* で設定されるDebianその他のレポジトリー上のパッケージをaptとdpkgのコマンドを使ってインストールします。

Debian sidには二万を越える数のパッケージがありますから、使いたいプログラムを apt-cache or with Debian Package Search GUI application.。

パッケージをインストールするにはapt-get install <パッケージ名>のコマンドを使います。パッケージのインストールを参照。

Debian sidのレポジトリは一日に平均して4回ほどアップデートされるため、インストールできるパッケージのリストと現行バージョンの情報はあっという間に古くなってしまいます。そのためapt-get dist-upgradeの前にまず必ずapt-get update を実行して、サーバーのパッケージリストについてののデータを更新するすることが肝心です。

Debian をベースにした他のディストリビューションやレポジトリー、ソースやRPMパッケージなどの使用について

ソースコードからのインストールはサポートされません。どうしてもアプリケーションを自分でコンパイルしなくてはならないときはユーザーとして作成し、システムにはインストールせずにホームディレクトリに置いてください。CheckInstallの使用やalienといったツールを使ってRPMをdebにコンバートしてインストールすることなどもサポートされません。

Debian をベースにした他の有名な(またはそれほど有名でない)ディストリビューションには、Debian のパッケージを独自のレポジトリー用にパッケージしなおしているものもあって、しばしばファイルの置き場所のためシステムが不安定になったり、パッケージのネーミングやバージョンの違いのために依存関係が解決せず、インストールに失敗したりすることがあります。別バージョンのglibcを使っているためにアプリケーションが実行できないこともしばしば起こります。

Debian のレポジトリーから必要なソフトウェアパッケージをインストールしましょう。他のレポジトリーを使用して問題がおこっても多くの場合サポートが受けられません。

システムのアップグレード - dist-upgrade

aptosidのアップグレードではapt-get dist-upgrade が公式にサポートされている方法です。GUIのフロントエンドを使ったアップグレードはaptosidではサポートされていません。システムのアップグレード -- dist-upgrade

Xウィンドウの上からのdist-upgradeはサポートされていません。(KDEやXFCEなどの)ウィンドウマネージャか、またはVT (virtual terminal - 仮想ターミナル、Ctrl+Alt+F1, Ctrl+Alt+F2など)から'init 3'を実行することでXが終了し、安全にアップグレードできるようになります。

ネットワーク設定

Ceni というネットワーク設定ツールで手早くLANカードやワイアレスカードを設定できます。ワイアレス機能はネットワークのスキャンWEPまたはWPAを使った暗号化ができ、設定のためのツールやWPAサプリカントを起動します。DHCP(IPアドレスの自動設定)を使っている場合のLAN設定は簡単で、そうでない場合はIPアドレスやネットマスク、ネームサーバーなどを手動で設定します。

Ceniは Ceniまたはceni のコマンドで実行されます。まだインストールされていないときはapt-get install ceniのコマンドでインストールできます。

他のウィンドウマネージャ

KDE、XFCEとfluxboxはaptosidにデフォルトで搭載されています. 現在のところ aptosid では Gnome はサポートされていません。aptosid フォーラムや wiki、IRCチャットなどで、Gnome の経験のあるユーザーからの援助を受けることはできるかもしれませんが、そうでない時は自力で頑張ってください。

日本語サポート

aptosid リリースにはささやかながら日本語のサポートが組み込まれています。

ライブCDの起動の際にF4を押して「日本語」を選ぶか、または起動オプションにlang=jaを加えることで、日本語ロケールがデフォルトになり、インストーラなどのメッセージが日本語表示されるようになります。(aptosid-kde-full のLive-DVDには日本語のフォントも入っていて、F4のオプションで日本語システムがスタートしますが、それ以外のLive-CDではデフォルトのロケールは設定されますが、起動後に日本語のフォントその他をインストールする必要があります。)

最低限の日本語入力環境の設定 (ibus)

(rootとして)
apt-get update
apt-get install ttf-kochi-gothic
apt-get install im-switch ibus-anthy ibus-qt4 ibus-gtk
im-switch -s ibus

注意：日本語環境のインストールの前にネットワークの設定が必要です

詳しくは日本語サポートのウィキページをご覧ください。
