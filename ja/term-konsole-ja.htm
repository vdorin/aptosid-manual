ターミナル/コンソールとは

ターミナル、またはコンソールとは、さまざまなコマンドを入力し、実行することでLinuxのOSと直接対話することのできるプログラムです。 他の呼び名としてしばしば 'shell' または 'command line'も使われます。 KDEではkonsoleというプログラムが用意されています。 ターミナルは非常に強力な道具として基本的な使い方を知っておく価値があります。

aptosidではターミナル/コンソールはKメニューの横にモニターのアイコンで表示されています。 えらんだテーマによってはアイコンの中に貝の絵が入っているかもしれません。同じアイコンをKメニューの"System"の中から選ぶこともできます。

ターミナルのウィンドウを開くと次のようなプロンプトが表示されます:

username@hostname:~$

usernameのところにあなたのログインネームが表示されているはずです。 ~ は今あなたのホーム・ディレクトリにいること、$はユーザーの権利でログインしていることを示しています。 最後にカーソルが表示されています。 これが実行したいコマンドを入力するためのコマンドラインです。

実行にあたってrootの権利を必要とするコマンドがたくさんあります。 そのためにはコマンドラインにsu とタイプしてEnterを押してください。 rootパスワードを聞かれますのでタイプしてEnterを押してください。 (パスワードの入力中はタイプした文字は表示されません)

もしパスワードが正しければ次のようなプロンプトが表示されます。

root@hostname:/home/username#

警告: rootとしてログインしている間、重要なファイルの消去といった危険なコマンドもそのまま実行されます。 システムを害する可能性も十分にありますから、どんなコマンドを実行しようとしているか完全に把握したうえで実行してください。

プロンプトの$マークが# に代わることに注目してください。 ターミナル/コンソール上ではいつでも# はrootの権利でログインしていることを示します。 このマニュアルでは$と#のマークの前に表示されるプロンプトを省略します。 次のコマンドは:

# apt-get install something

ターミナルを開き,(suxで)rootになり,コマンドを#のプロンプトに入力するという意味です。 (#をタイプしないように)

時にコンソールの画面がおかしくなってしまうことがあります。 そういう時は:

reset

とタイプしてEnterを押してください。

ターミナル/コンソールの表示が歪んで見えるときがあります。 そういうときはctrl+l, を押してターミナル・ウィンドウの再描画をすることで直ることがあります。 表示の歪みはncoursesのインターフェイスを使っているプログラム(irssiなど)で起こることが多いです

ターミナル/コンソールはときに凍ってしまって何の反応もしないように見えるのに、実際は動いていてタイプされたコマンドが実行されている、ということがあります。 これは誤ってctrl+sを押して締まったときに起こり、その場合 ctrl+qを押してターミナルのロックを解除することで直ります。

コンソールの user:~$ と root:# の プロンプト:の色分け

コンソールのプロンプトを色分けすることでuser~$のつもりでうっかりroot # として破滅的なコマンドを実行してしまったりすることを防いだり、マーカーとして100行ほど前に実行したコマンドもすぐ見付らるようしたりすることができます。

デフォルトでは user~$ と root# のプロンプトは同じ色ですが、どちらの色も簡単に変えることができます。

基本色の色指定コード:

(00;XXのシンタックス)
[00;30] Black   黒
[00;31] Red     赤
[00;32] Green   緑
[00;33] Yellow  黄色
[00;34] Blue    青
[00;35] Magenta マゼンタ
[00;36] Cyan    シアン
[00;37] White   白
[[00;XX] を [01;XX] に変えるとカラーバリエーションが得られます].
ユーザーとしての ~$ プロンプトの色を変えるには:

$ プロンプトのユーザーとして、お好みのテキストエディタで:

$ <editor> ~/.bashrc

39行目あたりの次のようなコメント行の先頭の # を消して有効なコマンドにします:

force_color_prompt=yes

53行目あたりの(一例として) 01;32m, のような色指定コード文字列のある行を探してお好み色のコードに変えます。

例として シアン [01;36m\]の user~:$ プロンプトにするには、3箇所の[01;XXm\] を編集して次のようにします:

PS1='${debian_chroot:+($debian_chroot)}\[\033[01;36m\]\u@\h\[\033[01;36m\]:\[\033[01;36m\]\w\[\033[00m\]\$ '

新しい設定は次回のターミナルログインの時から有効になります。

root としての # プロンプトの色を変えるには:
su
<mcedit> /root/.bashrc

39行目あたりの次のようなコメント行の先頭の # を消して有効なコマンドに>します:

force_color_prompt=yes

53行目あたりの(一例として) 01;32m, のような色指定コード文字列のある行>を探してお好み色のコードに変えます。

例として 赤 [01;31m\]の root:# プロンプトにするには、3箇所の[01;XXm\] を編集して次のようにします:

PS1='${debian_chroot:+($debian_chroot)}\[\033[01;31m\]\u@\h\[\033[01;31m\]:\[\033[01;31m\]\w\[\033[00m\]\$ '

新しい設定は次回のターミナルログインの時から有効になります。

ターミナルの背景色を変えるには

コンソールターミナルの背景色やフォントを変えるにはターミナルプログラムのメニューのオプションから探してください。

色設定に凝りはじめると限りなく様々な設定ができますが、あまり凝らずになるべくシンプルな設定にしておくようにおすすめします。

About Xapps whilst in X as root and sux2 as an alias

While the common command for becoming root is <su>, using kdesu <Application> or gksu <Application> will allow you run GUI / X11 applications from the desktop and allow root to start graphical applications.

For those used to using sux , (sux has been broken by a 'login' security update for a long time), will find that sux2, as an alias, is a good option in the meantime otherwise use, as $user:

$ suxterm

In most cases, using an Xapp while in root is to take advantage of a GUI editor like kate, kwrite, mousepad or leafpad to name just a few. X while in root is also required for the GUI partitioning application gparted. (Also see sudo).

Non-gui applications like mcedit, vim, vi, or emacs are highly recommended as editors, with mcedit being one of the intuitive editors of the non-gui applications for use by first time users of a text editor while in root with X. mcedit, vim, vi or emacs are possibly your only choices of an editor while not in X. (mc is an ideal file browser while not in X).

KDE options

To start kdesu in KDE:

Alt+F2

or right-click on the desktop and choose:

Run Command

then:

kdesu <Application>

Some KDE applications require dbus-launch in front of the application:

dbus-launch <Application>

Xfce options

To start Run Command in Xfce:

Alt+F2

or right-click on the desktop and choose:

Run Command

then:

gksu <Application>

Other Desktop Window Manager options

Another keyboard option also generic to all major Desktop Managers is:

Alt+F2

then:

su-to-root -X -c <Application>

Terminal/konsole options

To run kdesu in a terminal:

/usr/lib/kde4/libexec/kdesu

To run gksu in a terminal:

/usr/bin/gksu

Another option is ssh -X root@localhost otherwise use su-to-root -X -c :

ssh -X root@localhost

Whilst ssh is installed by defualt it may not be enabled. To enable ssh, as root: 

/etc/init.d/ssh start

else:

su-to-root -X -c <application>

Setting up an alias in a the terminal for kdesu, gksu, su-to-root -X -c or ssh -X root@localhost called sux2

Typing a long command line in the terminal all the time is not an ideal, so the solution is to set up an terminal/konsole alias called sux2.

Note: In each of the following examples the command to activate the alias in a terminal will be:

sux2 <application>
Setting an alias

The use of mcedit as $user in the following examples is deliberate. Feel free to use any editor you wish to use:

$ mcedit ~/.bashrc

Note:To enable any changes you have made you need to close the terminal/konsole and restart it.

Add the following code to the file ~/.bashrc by appending the code to the end of the file:

Setting kdesu an alias

alias sux2='/usr/lib/kde4/libexec/kdesu'

Setting gksu an alias

alias sux2='/usr/bin/gksu'

Setting su-to-root -X -c as an alias

alias sux2='su-to-root -X -c'

Setting ssh -X root@localhost as an alias

alias sux2='ssh -X root@localhost'

You may want to implement a ~/.bash_alias file to contain all the aliases that you find useful. Therefore make a file called ~/.bash_alias. Read the ~/.bashrc file and /usr/share/doc/bash-doc/examples:

apt-get install bash-doc

sudo はサポートされていません

ハードディスクにインストールされたシステムでは sudo コマンドはデフォルトでサポートされていません。ライブCDではrootのパスワードがないのでsudo を使っても問題がありません。sudo をサポートしない理由として、侵入者がもし一人のユーザーのパスワードを手にしても、すぐにスーパーユーザーの権利でシステムを破壊することができなくなるからです。

sudo のもう一つの問題点として、ユーザーの設定でスーパーユーザー用のコマンドを実行しているうちに、ファイルの権利設定を知らない間に変更してしまうことがあることです。このためにその後コマンドをユーザーとして実行できなくなってしまうような例もあります。sudo の代わりにsux2, kdesu, gksu または su-to-root -X -c or ssh -X root@localhost などを使いましょう

rootになることの意味

警告: rootとしてログインしている間、重要なファイルの消去といった危険なコマンドもそのまま実行されます。 システムを害する可能性も十分にありますから、どんなコマンドを実行しようとしているか完全に把握したうえで実行してください。

E-mailを送ったり計算表を作ったり,ネットのサーフィングなど普通のユーザーの権利で日々使っているプログラムをrootとしてターミナル/コンソールから実行することはどんな場合であれ絶対に避けるべきです。

コマンドライン・ヘルプ

ちゃんとあります。 大抵のLinuxのコマンド/プログラムはコマンドラインから呼び出せる"man page"または"manual page"という名のマニュアルを備えています。 呼び方は:

$ man "command"

or

$ man -k <keyword>

これでそのコマンドのマニュアルページが読めます。 カーソルキーで上下にスクロールできます。 試しに次のコマンドを実行してみましょう:

$ man apt-get

マニュアルページから抜け出すには qをタイプしてください。

もう一つの便利なコマンドに "apropos" があります。 これはコマンドの名前やシンタックスをを正確に思い出せない時にマニュアルページをサーチするコマンドです。 試しに次のコマンドを実行してみましょう:

$ apropos apt-

パッケージマネージャ'apt'のすべてのコマンドがリストとして表示されます。 この'apropos'は非常に強力なツールで使い方のすべてを紹介するスペースはありませんので、詳しいことはマニュアルページを読んでください。
