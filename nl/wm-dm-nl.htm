Dit pakket bevat een variatie van handige plugins voor Konqueror, de file manager, web browser en document weergever voor KDE. Vele van deze plugins zullen in Konqueror's Tools menu verschijnen.

apt-get install konq-plugins

KDE Desktop Search - (Nepomuk and Strigi)

KDE's Nepomuk semantische desktop zoekmachine is niet aangeschakeld op de aptosid-kde.iso. Om aan te schakelen, doe het volgende:

Installeer de virtuoso-minimal en strigi-client pakketten:

apt-get install virtuoso-minimal strigi-client

Navigeer naar System Settings > System Administration > Startup and Shutdown >Service Manager, en markeer het vakje voor Nepomuk Search Module, en klik dan op Apply.

Navigeer naar System Settings > Workspace Appearance and Behavior > Desktop Search > Basic Settings, en markeer het vakje voor Enable Nepomuk Semantic Desktop and Enable Strigi Desktop File Indexer, en klik dan op Apply. Opmerking : De hoeveelheid geheugen dat wordt gereserveerd voor de Nepomuk database kan worden aangepast in de Advanced Settings tab.

De KDE sessie moet worden herstart om het indexeren effectief te laten starten. Je kan verwachten dat de eerste indexatie lang kan duren. Voor meer informatie over Nepomuk .

Loginproblemen voor KDE

Normaal gesproken wordt de inhoud van de /tmp directory verwijderd bij elke boot. Sommige directories en sockets die de X-server nodig heeft, worden dus ook verwijderd.

Gewoonlijk worden die door het script 'x11-common' voor X-Org tijdens het boot-proces gewoon aangemaakt.

Het kan zijn dat dit script niet wordt aangeroepen tijdens het boot-proces. Om deze links dan toch te maken, typ je in de console: 

# X-ORG: # dpkg-reconfigure x11-common

KDE heeft 5% ruimte nodig op de partitie waar de directory /tmp ligt, dit omdat er tijdelijke bestanden tijdens het inloggen worden aangemaakt. Dus als je partitie 95% vol is, kun je niet inloggen met KDE en kom je terug in terminal mode (tty).

Hetzelfde geldt voor kdm looping welke je niet toestaat om in te loggen. Een oplossing is om via tty (terminal) in te loggen om ruimte te creëren, door het verwijderen van applicaties / bestanden die je niet meer gebruikt.

Een ander alternatief is een andere X window manager gebruiken die niet zoveel ruimte van het systeem vereist ( bijvoorbeeld fluxbox is ook standaard geinstalleerd bij een aptosid install), of doe een chroot gebruikmakend van de aptosid live-CD/DVD om de partitie op te schonen, zodat je wel tijdens het booten KDE kan opstarten.

Maximaal een 85% volle partitie wordt aangeraden om KDE te kunnen gebruiken, zodat deze nog 15% ruimte heeft in de /tmp directory voor zijn tijdelijke bestanden.

Installeren van aptosid KDE Art en Thema's

Om de laatste aptosid-art te installeren op een bestaande aptosid installatie voer je het volgende uit:

apt-get install aptosid-art-kde-xxxx aptosid-art-wallpaper-xxxx
(Where xxxx is the name of the release for example aptosid-art-kde-momos)

Dit installeert de aptosid achtergrond en thema's

Voor het veranderen van de achtergrond:

Rechtermuisklik op je desktop en kies Desktop Settings. Onder de kop Wallpaper is er een subkopje genoemd Picture welke je een dropdown lijst geeft met plaatjes erin waaruit je kunt kiezen. Je kunt ook op de Browse knop klikken zodat je een eigen plaatje kunt uit kiezen voor je PC.

Om het inlogscherm te veranderen:

Om het inlogscherm te veranderen, moet je eerst systemsettings openen met root rechten:

Alt  + F2 (om krunner te starten)

kdesu systemsettings

Klik nu op de Advanced tab en klik dan op Login Manager, ga naar de Thema tab en kies je gewenste thema. Om de nieuwe login manager te activeren moet je de computer opnieuw opstarten.

Installeren van andere desktop omgevingen naast de bestaande desktop:

Wanneer je een andere desktop omgeving installeert naast je huidige installatie, (bijvoorbeeld je hebt aptosid-kde.iso geinstalleert en je wilt XFCE of LXDE desktop omgeving erbij installeren), dan zal er waarschijnlijk een display manager (dm) erbij geinstalleerd zijn of je moet deze nog zelfs installeren (gdm, slim of een andere dm pakket).

Je zal echter met een default Debian runlevel configuratie achterblijven, wat wil zeggen dat je manueel X moet stoppen in runlevel 3 alvorens een dist-upgrade te doen.

De oplossing hiervoor is als volgt:

apt-get update
apt-get install --reinstall distro-defaults
update-rc.d <dm> remove
update-rc.d <dm> defaults

Voorbeelden voor update-rc.d <dm> defaults en let op de punt . :

update-rc.d kdm start 24 5 . stop 01 0 1 2 3 4 6 .

update-rc.d gdm start 24 5 . stop 01 0 1 2 3 4 6 .

update-rc.d slim start 01 5 . stop 01 0 1 2 3 4 6 .

Desktop 'bevriest'

In een dergelijke situatie heb je niet altijd de reset button nodig. Dit kan tenslotte je bestandssysteem beschadigen of leiden tot dataverlies. In elk geval zal het bestandssysteem niet 'schoon' zijn na een harde reboot ("filesystem not clean")

Probeer eerst naar een tekstconsole te gaan met: alt-ctrl-F1 of start de X-server opnieuw op met: alt-ctrl-backspace. Ook als geen van deze twee opties werkt, is er nog steeds hoop:

De SysRq-toets (print-toets, aan de rechterbovenkant van het toetsenbord) kan je helpen om een 'bevroren' desktop netjes te rebooten.

De hieronderstaande toetsencombinaties zijn mogelijk:
* alt+sysrq+r (geeft je de controle terug over je toetsenbord)
* alt+sysrq+s (leidt tot een sync)
* alt+sysrq+e (stuurt term naar alle processen behalve init)
* alt+sysrq+i (stuurt kill naar alle processen behalve init)
* alt+sysrq+u (bestandssysemen worden slechts readonly gemount, voorkomt fsck tijdens reboot)
* Alt+sysrq+b (reboot het systeem; zonder de vorige stappen zou dit een harde reboot zijn).

Geef elke stap een paar seconden de tijd om uitgevoerd te worden; het beëindigen van alle processen duurt bijvoorbeeld wel een tijdje. De hierboven genoemde letters kun je onthouden met het volgende, engelse, ezelsbruggetje:
"Reboot System Even If Utterly Broken"

Of met het volgende ezelsbruggetje:
"Raising Skinny Elephants Is Utterly Boring"
