Rebuilding fstab and creating mount points

By default aptosid uses uuid in your fstab when you install.

Για να εμφανίσετε μια καινούργια κατάτμηση (partition) (π.χ. sda6 ή sdb7), που δεν εμφανίζεται στο fstab ή δεν γίνεται mount, σε ένα τερματικό σαν απλός χρήστης ($), πληκτρολογήστε την ακόλουθη εντολή:

 ls -l /dev/disk/by-uuid

Θα δείξει τα εξής (τα έντονα γράμματα είναι μόνο για τους σκοπούς του παραδείγματος):

lrwxrwxrwx 1 root root 10 2007-05-27 23:42 348ea9e6-7879-4332-8d7a-915507574a80 -> ../../sda4
lrwxrwxrwx 1 root root 10 2007-05-27 23:42 610aaaeb-a65e-4269-9714-b26a1388a106 -> ../../sda2
lrwxrwxrwx 1 root root 10 2007-05-27 23:42 857c5e63-c9be-4080-b4c2-72d606435051 -> ../../sda5
lrwxrwxrwx 1 root root 10 2007-05-27 23:42 a83b8ede-a9df-4df6-bfc7-02b8b7a5f1f2 -> ../../sda1
lrwxrwxrwx 1 root root 10 2007-05-27 23:42 ad662d33-6934-459c-a128-bdf0393e0f44 -> ../../sda6

Σε αυτό το παράδειγμα, το ad662d33-6934-459c-a128-bdf0393e0f44 είναι η χαμένη καταχώρηση. Το επόμενο βήμα είναι να βάλουμε το UUID τμήμα (partition) στο etc/fstab. Για να το προθέσετε στο αρχείο fstab χρησιμοποιήστε έναν κειμενογράφο (όπως το kate ή το kwrite) με δικαιώματα υπερχρήστη (root privileges):

# <device file system> <mount point>   <type>  <options>       <dump> <pass>
UUID=ad662d33-6934-459c-a128-bdf0393e0f44 /media/disk1part6 ext4	auto,users,exec	0 2

Ένα ακόμη παράδειγμα:

lrwxrwxrwx 1 root root 10 2007-05-28 13:18 30ebb8eb-8f22-460c-b8dd-59140274829d -> ../../sdb8
lrwxrwxrwx 1 root root 10 2007-05-28 13:18 387d6d4b-4508-4b8e-8ed2-76998f41dae4 -> ../../sdb1
rwxrwxrwx 1 root root 10 2007-05-28 13:18 7014f66f-6cdf-4fe1-83da-9cab7b6fab1a -> ../../sdb5
lrwxrwxrwx 1 root root 10 2007-05-28 13:18 8f042ead-259f-4df0-98ec-3343080396c5 -> ../../sdb6
lrwxrwxrwx 1 root root 10 2007-05-28 13:18 94B0AE63B0AE4B94 -> ../../sda2
lrwxrwxrwx 1 root root 10 2007-05-28 13:18 A61820AA18207B85 -> ../../sda1
lrwxrwxrwx 1 root root 10 2007-05-28 13:18 f28725d6-b7b5-4207-8476-36efe1a903ce -> ../../sdb9
lrwxrwxrwx 1 root root 10 2007-05-28 13:18 f855c263-2521-48d3-8ec9-d2d2b69b6635 -> ../../sda3
rwxrwxrwx 1 root root 10 2007-05-28 13:18 f9aa4027-ecdd-4a86-84e2-df2ef73fe14e -> ../../sdb7

Σε αυτή την περίπτωση το f9aa4027-ecdd-4a86-84e2-df2ef73fe14e είναι η χαμένη καταχώρηση και την προσθέτουμε στο /etc/fstab ως εξής:

# <device file system> <mount point>   <type>  <options>       <dump> <pass>
UUID=f9aa4027-ecdd-4a86-84e2-df2ef73fe14e /media/disk2part7 ext4	auto,users,exec	0 2

Creating new mount points

Note: A mount point name, as noted in fstab, needs to have an existing directory. aptosid creates these directories during the installation process under /media and they are named diskXpartX.

If you have manipulated the partition table after the initial installation and assuming you have already altered fstab, (for example, 2 new partitions have been created), the directory for each mount point will not exist and it needs to be manually created.

Example:

First, as root, confirm the existing mount points:

cd /media
ls

It should return the existing mount points, for example:

disk1part1 disk1part3 disk2part1

Staying in /media, create the mount points of the new partitions:

mkdir disk1part6
mkdir disk2part7

To test or use partitions immediately:

mount /dev/sda6 /media/disk1part6
mount /dev/sda6 /media/disk2part7

Upon a reboot of the computer the filesystems will be mounted automatically. Read:

man mount

Επισκόπηση: UUID, Ετικέτες (Labels) Κατατμήσεων και fstab

Η επίμονη ονομασία (persistent naming) block συσκευών κατέστη δυνατή με την εισαγωγή του udev και έχει μερικά πλεονεκτήματα σε σχέση με την ονομασία βάση του bus (bus-based naming).

Ενώ οι διανομές Linux και το udev εξελίσσονται και η αναγνώριση υλικού (hardware detection) γίνεται πιο αξιόπιστη, υπάρχουν επίσης πολλά νέα προβλήματα και αλλαγές:
1) Εάν έχετε περισσότερο από ένα ελεγκτή (controller) για σκληρούς δίσκους sata/scsi ή ide , η σειρά με την οποία προστίθενται είναι τυχαία. Αυτό μπορεί να έχει σαν αποτέλεσμα ονόματα συσκευών όπως hdX και hdY να ανταλλάσσονται τυχαία μεταξύ τους σε κάθε εκκίνηση. Το ίδιο ισχυεί για sdX και sdY. Η επίμονη ονομασία (persistent naming) σας επιτρέπει να μην ανησυχείτε γιαυτό.
2) Με την εισαγωγή της νέας libata pata υποστήριξης, όλες οι ide hdX συσκευές θα γίνουν sdX συσκευές, σε κάποια στιγμή στο μέλλον. Και πάλι λόγω της επίμονης ονομασίας, ούτε που θα το προσέξετε.
3) Υπολογιστές με sata και ide ελεγκτές είναι κοινή την σήμερον ημέρα. Με τις libata αλλαγές που αναφέρθηκαν πιο πάνω, το πρώτο πρόβλημα θα γίνει ακόμη πιο συχνό γιατί και οι sata και οι ide σκληροί δίσκοι θα έχουν τα ίδια sdX ονόματα.

Εκ προεπιλογής το aptosid θα χρησιμοποιήσει uuid μέσα στο fstab όταν κάνετε εγκατάσταση

Υπάρχουν και άλλοι λόγοι αλλά αυτοί είναι οι κυριότεροι τώρα και στο εγγύς μέλλον. Γιαυτό το aptosid σας ενθαρρύνει να αλλάξετε τις ρυθμίσεις σας και να χρησιμοποιείτε επίμονα ονόματα (persistent naming schemes).

Τα τέσσερα διαφορετικά καθεστώτα (schemes) για επίμονη ονομασία (persistent naming):

1. Επίμονη ονομασία με βάση το UUID

UUID σημαίνει Universally Unique Identifier (Καθολικά Μοναδικό Αναγνωριστικό) και είναι ένας μηχανισμός που δίνει σε κάθε σύστημα αρχείων ένα μοναδικό αναγνωριστικό. Είναι σχεδιασμένο έτσι ώστε συγκρούσεις (collisions) να είναι απίθανες. Όλα τα συστήματα αρχείων Linux (και το swap) υποστηρίζουν UUID. FAT και NTFS συστήματα αρχείων δεν υποστηρίζουν UUID, αλλά εξακολουθεί να τους δίνετε ένα μοναδικό αναγνωριστικό με βάση το uuid:

$ /bin/ls -lF /dev/disk/by-uuid/
total 0
lrwxrwxrwx 1 root root 10 Oct 16 10:27 2d781b26-0285-421a-b9d0-d4a0d3b55680 -> ../../sda1
lrwxrwxrwx 1 root root 10 Oct 16 10:27 31f8eb0d-612b-4805-835e-0e6d8b8c5591 -> ../../sda7
lrwxrwxrwx 1 root root 10 Oct 16 10:27 3FC2-3DDB -> ../../sda6
lrwxrwxrwx 1 root root 10 Oct 16 10:27 5090093f-e023-4a93-b2b6-8a9568dd23dc -> ../../sda2
lrwxrwxrwx 1 root root 10 Oct 16 10:27 912c7844-5430-4eea-b55c-e23f8959a8ee -> ../../sda5
lrwxrwxrwx 1 root root 10 Oct 16 10:27 B0DC1977DC193954 -> ../../sdb1
lrwxrwxrwx 1 root root 10 Oct 16 10:27 bae98338-ec29-4beb-aacf-107e44599b2e -> ../../sdb2

Όπως βλέπετε οι fat και ntfs κατατμήσεις έχουν μικρότερα ονόματα (sda6 και sdb1), αλλά εξακολουθούν να έχουν uuid.

2. Επίμονη ονομασία με βάση την ΕΤΙΚΕΤΑ (label)

Σχεδόν κάθε σύστημα αρχείων μπορεί να έχει ετικέτα. Όλες οι κατατμήσεις σας που έχουν μια ετικέτα είναι καταχωρημένες στον /dev/disk/by-label φάκελο:

$ ls -lF /dev/disk/by-label
total 0
lrwxrwxrwx 1 root root 10 Oct 16 10:27 data -> ../../sdb2
lrwxrwxrwx 1 root root 10 Oct 16 10:27 data2 -> ../../sda2
lrwxrwxrwx 1 root root 10 Oct 16 10:27 fat -> ../../sda6
lrwxrwxrwx 1 root root 10 Oct 16 10:27 home -> ../../sda7
lrwxrwxrwx 1 root root 10 Oct 16 10:27 root -> ../../sda1
lrwxrwxrwx 1 root root 10 Oct 16 10:27 swap -> ../../sda5
lrwxrwxrwx 1 root root 10 Oct 16 10:27 windows -> ../../sdb1

Αν και είναι ευκολότερο να αναγνωρίσετε τις ετικέτες, θα πρέπει να προσέχετε ιδιαίτερα να μην υπάρχουν δυο ετικέτες με το ίδιο όνομα.

Μπορείτε να αλλάξετε τις ετικέτες των συστημάτων αρχείων σας με τις παρακάτω εντολές:

* swap: Δημιουργείστε ένα νέο swapspace έτσι: mkswap -L <ετικέτα> /dev/XXX
* ext2/ext3/ext4: e2label /dev/XXX <ετικέτα>
* jfs: jfs_tune -L <ετικέτα> /dev/XXX
* xfs: xfs_admin -L <ετικέτα> /dev/XXX
* fat/vfat: Δεν υπάρχει εργαλείο για να αλλάξετε την ετικέτα χρησιμοποιώντας Linux,
αλλά όταν δημιουργείτε το σύστημα αρχείων, χρησιμοποιήστε mkdosfs -n <ετικέτα> <άλλες επιλογές>.
Μπορείτε επίσης να αλλάξετε την ετικέτα ενός υπάρχοντος συστήματος αρχείων με την χρήση.
* ntfs: ntfslabel /dev/XXX <ετικέτα> ή αλλάξτε το χρησιμοποιώντας Windows.

Προσέξτε: Οι ετικέτες πρέπει να είναι μοναδικές για να πετύχει αυτό, ισχύει εξίσου για USB/firewire sticks και σκληρούς δίσκους. Η LABEL=/ UUID= σύνταξη προτιμάται σε σχέση της /dev/disk/by-*/ για UN*X κατατμήσεις

3. Επίμονη ονομασία με βάση το id

by-id δημιουργεί ένα μοναδικό όνομα, εξαρτώμενο από τον σειριακό αριθμό του υλικού.

4. Επίμονη ονομασία με βάση το μονοπάτι (path)

by-path δημιουργεί ένα μοναδικό όνομα εξαρτώμενο από το συντομότερο φυσικό μονοπάτι (σύμφωνα με το sysfs). Οι 2 τελευταίες μέθοδοι που αναφέρθηκαν περιέχουν strings για να υποδείξουν σε πιο υποσύστημα ανήκουν και γιαυτό δεν είναι κατάλληλες για την επίλυση των προβλημάτων που αναφέρθηκαν στην αρχή του άρθρου αυτού. Δεν θα συζητηθούν περαιτέρω εδώ.

Ενεργοποιώντας επίμονη ονομασία

Έχοντας επιλέξει την μέθοδο ονομασίας που θέλετε να χρησιμοποιήσετε, ας ενεργοποιήσουμε τώρα την επίμονη ονομασία για το σύστημα σας:

Μέσα στο fstab

Η ενεργοποίηση επίμονης ονομασίας στο /etc/fstab είναι εύκολη. Απλά αντικαταστήστε το όνομα της συσκευής στην πρώτη στήλη με το νέο μόνιμο όνομα. Στο παράδειγμα θα αντικαταστήσω το /dev/sda7 με ένα από τα ακόλουθα:

/dev/disk/by-label/home ή
/dev/disk/by-uuid/31f8eb0d-612b-4805-835e-0e6d8b8c5591

Κάντε το αυτό για όλες τις κατατμήσεις μέσα στο fstab αρχείο σας.

Αντί να δώσετε την συσκευή ρητά, θα μπορούσατε να δηλώσετε το σύστημα αρχείων το οποίο θα προσαρτηθεί (mount) με το UUID του ή την ετικέτα του, γράφοντας LABEL=<ετικέτα> ή UUID=<uuid>, για παράδειγμα:

LABEL=Boot

ή

UUID=3e6be9de-8139-11d1-9106-a43f08d823a6
