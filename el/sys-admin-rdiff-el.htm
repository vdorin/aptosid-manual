Κάνοντας αντίγραφα ασφαλείας (Backing-Up) του συστήματος με το rdiff-backup

Το rdiff-backup είναι ένα εργαλείο που χρησιμοποιείται για να κάνετε backup τα αρχεία σας. (Μπορεί να λειτουργήσει σε διάφορα *nix περιβάλλοντα).

Εκτελέστε τις εντολές στην κονσόλα σαν διαχειριστής (root) εκτός και αν αναφέρεται κάτι άλλο.

*Πολύ καλό για να επαναφέρετε συστήματα που έχουν πρόβλημα μετά από dist-upgrades (ενημέρωση συστήματος), ενημέρωση πυρήνα κτλ. (και επίσης για να επανακτήσετε μεμονωμένα αρχεία).
*κάνει back-up ότι άλλαξε όπως κάνει και το rsync (έτσι το κάθε backup δεν παίρνει πολύ χρόνο).
*κρατά ημερολόγιο με τις αλλαγές (πράγμα που σημαίνει ότι μπορείτε να επανακτήσετε ένα αρχείο που σβήσατε πριν από τρεις εβδομάδες!)
*κάνει ασφαλή backup μέσω δικτύου (με χρήση ssh).
*κάνει backup κατατμήσεις ενώ είναι προσαρτημένες (mounted) (οπό ταν είναι εύκολο να αυτοματοποιήσετε ένα καθημερινό backup ... δεν χρειάζεται αποπροσάρτηση).
*μπορεί να ανακτήσει τα πάντα εάν ο δίσκος σας "πεθάνει" και πρέπει να αγοράσετε καινούργιο.
*μπορεί να κάνει backup μεγάλα δίκτυα (linux είναι εντάξει, windows είναι πιο δύσκολο) και χρησιμοποιείται από εταιρίες.
*είναι μια εφαρμογή της γραμμής εντολών (command-line-driven app) και γιαυτό είναι πολύ καλή για αυτούς που τους αρέσει να κάνουν πράγματα, όπως αυτοματοποιημένα backups με ένα ισχυρό τρόπο (πχ ένα bash script το οποίο καλείται από το cron).
*Θυμάται και χειρίζεται τις ιδιοκτησίες (ownerships) και δικαιώματα (permissions) των αρχείων. Επίσης χειρίζεται συμβολικούς συνδέσμους (symbolic links) (και όλα τα παρόμοια πράγματα) έτσι όταν ανακτάτε το σύστημα, τα πράγματα είναι ακριβώς όπως ήταν.

Τι θα χρειαστείτε

Το rdiff-backup κρατάει ένα ολόκληρο (ασυμπίεστο) αντίγραφο των αρχείων που κάνετε backup και κρατάει επίσης ένα ημερολόγιο (history) (incremental backups). Αυτό σημαίνει ότι ο χώρος που χρειάζεται για backup πρέπει να είναι μεγαλύτερος από αυτά που κάνετε backup. Εάν κάνετε backup 100 gigs (giga bytes), μπορεί να χρειαστείτε 120 gigs χώρο (κατά προτίμηση σε ξεχωριστό σκληρό δίσκο!).

Πως να το ρυθμίσετε

Ας πούμε ότι ο υπολογιστής σας έχει τα πιο κάτω:
* ένα 100 gig σκληρό δίσκο (sda), ο οποίος χρησιμοποιείται, με σημεία προσάρτησης (mounts) sda1 που είναι η root κατάτμηση, sda5 που χρησιμοποιείται για την φύλαξη μουσικών αρχείων και άλλων αρχείων και sda6 για swap.
* ένα εξτρα 200 gig σκληρό δίσκο (sdb), ο οποίος δεν χρησιμοποιείται, με ένα σημείο προσάρτησης sdb1. θα χρησιμοποιήσουμε αυτόν για τα backup μας.
* IP διεύθυνση 192.168.0.1

Το πρώτο πράγμα που πρέπει να κάνετε είναι να εγκαταστήσετε το rdiff-backup:

# apt-get install rdiff-backup

Τώρα, αν και μπορείτε να κάνετε backup οποιοδήποτε κατάλογο, θέλουμε να σας δείξουμε πως να κάνετε backup ολόκληρες κατατμήσεις. Θέλουμε να κάνουμε backup την sda1 και την sda5 (δεν θέλουμε να κάνουμε backup την sda6) οπό ταν κάνουμε μερικούς καταλόγους για να φυλάξουμε τα δεδομένα:

# mkdir -p /media/sdb1/rdiff-backups/192.168.0.1/root
# mkdir -p /media/sdb1/rdiff-backups/192.168.0.1/sda5

Χρειάζεται να διευκρινίσετε διεύθυνση IP εάν θέλετε να χρησιμοποιήσετε αυτόν τον υπολογιστή για να κάνετε backup ένα άλλο υπολογιστή (καλύπτεται πιο κάτω).

Κάνοντας το backup

Το rdiff-backup χρησιμοποιεί την σύνταξη rdiff-backup πηγή προορισμός. Σημείωση: πάντα να δίνετε ονόματα καταλόγων και όχι ονόματα αρχείων.

Για να κάνετε backup το sda5, τρέξτε:

# rdiff-backup /media/sda5 /media/sdb1/rdiff-backups/192.168.0.1/sda5

Και για να κάνετε backup την root κατάτμηση, τρέξτε:

# rdiff-backup --exclude '/tmp/*' --exclude '/proc/*' --exclude '/sys/*' --exclude '/media/*/*' / /media/sdb1/rdiff-backups/192.168.0.1/root

Οποιαδήποτε "AF_UNIX path too long" λάθη (errors) μπορούν να αγνοηθούν. Μπορεί να χρειαστεί λίγη ώρα μιας και είναι η πρώτη φορά που η κατάτμηση θα γίνει backup και το rdiff-backup θα πρέπει να κάνει backup ολόκληρη την κατάτμηση (όχι μόνο τις διαφορές). Σημειώστε ότι δεν θέλουμε να κάνουμε backup το /tmp γιατί αυτό πάντα αλλάζει, ούτε το το /proc ή το /sys γιατί δεν περιέχουν πραγματικά αρχεία και επίσης δεν θέλουμε να κάνουμε backup τα σημεία προσάρτησης. Εάν κάνετε backup σημεία προσάρτησης, τότε θα κάνατε backup το sdb1 και θα μπορούσατε να μπείτε σε άπειρο βρόχο (infinite loop)! Μια λύση γιαυτό είναι να κάνετε ξεχωριστά backup τα σημεία προσάρτησης.

Τώρα ο λόγος που βάζετε '/proc/*' αντί του '/proc' είναι γιατί αυτό θα κάνει backup τον κατάλογο με το όνομα /proc , αλλά θα αγνοήσει ότι υπάρχει μέσα. Το ίδιο ισχύει για το /tmp , /sys και ακόμη περισσότερο για όλα τα ονόματα των σημείων προσάρτησης.

Με αυτόν τον τρόπο, εάν καταστρέψετε την root κατάτμηση σας και κάνετε μια πλήρη επαναφορά, τα /tmp, /proc, /sys και τα ονόματα τον σημείων προσάρτησης θα δημιουργηθούν (όπως θα έπρεπε). Εάν το /tmp δεν υπάρχει όταν ξεκινάει ο X , μπορεί να παραπονεθεί. (Δέστε την σελίδα εγχειριδίου (man page) για περισσότερα για τα --exclude και --include) .

Επαναφορά καταλόγων από τα backup

το rdiff-backup χρησιμοποιεί αυτή την σύνταξη:

rdiff-backup -r <από-πότε> <πηγή> <προορισμός>

Τώρα εάν κατά λάθος σβήσατε τον κατάλογο /media/sda7/photos, μπορείτε να τον ανακτήσετε γράφοντας:

# rdiff-backup -r now /media/sdb1/rdiff-backups/192.168.0.1/sda5/photos /media/sda5/photos

Η "-r now" επιλογή σημαίνει ανάκτησε από το πιο πρόσφατο backup. Εάν κάνατε τα πάντα backup περιοδικά (μέσω crontab, ας πούμε) και δεν συνειδητοποιήσατε ότι ο κατάλογος λείπει εδώ και μερικές μέρες, θα πρέπει να κάνετε την ανάκτηση από ένα backup μερικών ημερών (και όχι "now" γιατί το πιο πρόσφατο backup λέει ότι ο κατάλογος με τις φωτογραφίες δεν υπάρχει). Ή ίσος θέλετε απλά μια παλαιότερη έκδοση ενός αρχείου.

Εάν θέλετε να επαναφέρετε από τρεις μέρες πριν, τότε χρησιμοποιήστε το "-r 3D" αλλά, όπως λέει και η σελίδα εγχειριδίου (man page), σημειώστε:

Το "3D" αναφέρεται στην στιγμή 72 ώρες πριν το παρόν και αν δεν έγινε backup εκείνη την ώρα, το rdiff-backup επαναφέρει την κατάσταση που καταγράφτηκε για το προηγούμενο backup. Για παράδειγμα, για την πιο πάνω περίπτωση, εάν χρησιμοποιηθεί το "3D" και υπάρχουν μόνο backup από 2 μέρες και 4 μέρες πριν, ο κατάλογος θα επαναφερθεί στην κατάσταση στην οποία βρισκόταν πριν από 4 μέρες (γιαυτό πρέπει να το σκεφτείτε πριν κάνετε ανάκτηση).

Η χρήση της παρακάτω εντολής θα σας δείξει τις μέρες και τις ώρες που έγιναν τα backup για το sda5:

# rdiff-backup -l /media/sdb1/rdiff-backups/192.168.0.1/sda5
Επαναφορά κατατμήσεων

Μπορείτε να επαναφέρετε ολόκληρες κατατμήσεις (σημεία προσάρτησης), στο κάτω κάτω, ένα σημείο προσάρτησης είναι απλά ένας κατάλογος.

ΠΡΟΣΟΧΗ: Μην επαναφέρετε την root κατάτμηση ενώ χρησιμοποιείται (booted into it)! Με μια εντολή θα χάσετε όλα σας τα αρχεία σε όλες σας τις κατατμήσεις καθώς και όλα τα backup που βρίσκονται σε ξεχωριστό δίσκο!! Το rdiff-backup κάνει ακριβώς αυτό που του είπατε να κάνει. Εάν το backup για την root κατάτμηση έχει άδεια σημεία προσάρτησης, για να μπορέσει να επαναφέρει το σύστημα στην κατάσταση του backup, διαγράφει τα πάντα μέσα στα σημεία προσάρτησης όπως είναι και στο backup.

Για να επαναφέρουμε την sda5 από το πιο πρόσφατο backup, απλά γράφουμε:

# rdiff-backup -r now /media/sdb1/rdiff-backups/192.168.0.1/sda5 /media/sda5
Επαναφέροντας την root κατάτμηση

Η επαναφορά της root κατάτμησης δεν είναι τόσο απλή. Μην επαναφέρετε την root κατάτμηση ενώ είναι προσαρτημένη (δείτε την προειδοποίηση πιο πάνω). Είναι πολύ χρήσιμο να μπορείτε να επαναφέρετε την root κατάτμηση, γιατί έτσι εάν κάνετε κάποιο λάθος κατά την διάρκεια της εγκατάστασης/ενημέρωσης ή αν κάνετε κάποιο λάθος στην εγκατάσταση ενός νέου πυρήνα (κτλ), θα έχετε το κεφάλι σας ήσυχο γιατί θα μπορείτε να επαναφέρετε τα πράγματα όπως ακριβώς ήταν πριν και θα σας πάρει μόνο 20 λεπτά.

Ένας τρόπος να επαναφέρετε την root κατάτμηση, είναι να εκκινήσετε (boot) από μια άλλη linux κατάτμηση, αν έχετε μια πάνω στον σκληρό σας δίσκο. Τότε θα μπορείτε να επαναφέρετε την κατάτμηση που θέλετε μιας και δεν θα είναι προσαρτημένη σαν root. Μετά που θα επαναφέρετε την κατάτμηση, εκκινήστε από αυτή και θα είναι ακριβώς όπως ήταν όταν κάνατε το backup! Αυτή είναι με διαφορά η πιο εύκολη μέθοδος.

Ένας άλλος τρόπος επαναφοράς της root κατάτμησης, είναι να εκκινήσετε το aptosid-live cd και να κάνετε την επαναφορά από εκεί. Το rdiff-backup περιέχεται μέσα στο aptosid. Σε περίπτωση που η έκδοση του aptosid live-cd που έχετε δεν περιλαμβάνει το rdiff, μπορείτε να γράψτε στον grub (Bootoptions, (Cheatcodes)) το cheatcode "unionfs" και αυτό θα σημαίνει ότι μπορείτε να εγκαταστήσετε εφαρμογές στο live cd. Απλά εκκινήστε και δώστε τις ακόλουθες εντολές:

$ sudo su
# wget -O /etc/apt/sources.list http://aptosid.com/files/misc/sources.list
# apt-get update
# apt-get install rdiff-backup
Τώρα ας κάνουμε την επαναφορά:
# mount /dev/sda1 /media/sda1
# mount /dev/sdb1 /media/sdb1
# rdiff-backup -r now /media/sdb1/rdiff-backups/192.168.0.1/root /media/sda1

Σημείωση: Αν δεν έχετε ένα aptosid CD και το Live-CD σας υποστηρίζεται από το klik, μπορείτε να εγκαταστήσετε το rdiff-backup χρησιμοποιόντας το Klik και καλώντας:

$ sudo ~/.zAppRun ~/Desktop/rdiff-backup_0.13.4-5.cmg rdiff-backup -r now /media/sdb1/rdiff-backups/192.168.0.1/root /media/sda1

Σε οποιοσδήποτε κάνει backup την root κατάτμηση του, με πρόθεση να την επαναφέρει από το backup αν χρειαστεί, συνιστάται να δοκιμάσει την διαδικασία επαναφοράς. Δεν υπάρχει τίποτα χειρότερο από το να νομίζετε ότι όλα θα είναι εντάξει και μετά να αντιμετωπίσετε κάτι απρόσμενο κατά την διάρκεια ενός επείγον περιστατικού.

Εάν αλλάξατε τον σκληρό δίσκο ή αν τον διαμορφώσατε, ξανά ελέγξτε τα UUIDs, (ή τις Ετικέτες), στο /boot/grub/menu.lst (grub-legacy) or files in /etc/grub.d (grub2)/boot/grub/menu.lst (grub-legacy) or files in /etc/grub.d (grub2) και /etc/fstab, και αλλάξτε τις αναλόγως. Ένας εύκολος τρόπος απόκτησης των πληροφοριών για την αλλαγή των menu.lst και fstab αρχείων, εάν χρειαστεί, είναι σαν root:

Κάνοντας backup άλλους υπολογιστές

Μπορείτε να κάνετε backup άλλους υπολογιστές πάνω στον τοπικό (local) σας υπολογιστή, φτάνει ο τοπικός σας υπολογιστής να μπορεί να συνδεθεί στον άλλο υπολογιστή μέσω ssh (και φτάνει να έχετε χώρο στον σκληρό σας δίσκο). Ο ssh διακομιστής (server) (sshd) πρέπει να τρέχει στον απομακρυσμένο (remote) υπολογιστή. Οι άλλοι υπολογιστές δεν χρειάζεται να βρίσκονται στο τοπικό σας δίκτυο (lan), μπορούν να βρίσκονται οπουδήποτε στον κόσμο.

Ας υποθέσουμε ότι ο απομακρυσμένος υπολογιστής σας έχει τα ακόλουθα:
1) ένα 100 gig σκληρό δίσκο (sda) ο οποίος χρησιμοποιείται, με μοναδικά σημεία προσάρτησης:
2) sda1 χρησιμοποιείται για την root κατάτμηση,
3) sda5 στην οποία φυλάσσονται ορισμένα προσωρινά αρχεία τα οποία δεν θέλουμε να κάνουμε backup
4) και sda6 για swap
5) διεύθυνση IP 192.168.0.2

Σημείωση: και οι δυο 100 gig δίσκοι δεν μπορούν συνήθως να γίνουν back up πάνω στον 200 gig δίσκο χρησιμοποιώντας το rdiff-backup (γιατί δεν θα υπάρχει χώρος για επιπλέων αρχεία), αλλά αφού δεν κάνετε back up τον sda5 στον απομακρυσμένο υπολογιστή (και μιας και οι δίσκοι δεν είναι συνήθως 100% γεμάτοι, αν και δεν πρέπει να βασιστείτε σε αυτό) μπορείτε να υπολογίσετε ότι έχετε αρκετό χώρο. Κάθε φορά που τρέχει το rdiff-backup, περισσότερα αρχεία δημιουργούνται και αυτά πιάνουν όλο και περισσότερο χώρο.

Μπορείτε να πείτε στο rdiff-backup να κρατάει τα backup μόνο για ένα μήνα (την εντολή αυτή σας την δείχνουμε αργότερα) και αυτό θα χρειάζεται πιο λίγο χώρο παρά αν του πείτε να κρατάει τα backup για ένα χρόνο. Φυσικά αν θέλετε να κρατάτε τα δεδομένα ενός χρόνου, τότε θα πρέπει να έχετε και χώρο στον σκληρό σας δίσκο για αυτά τα δεδομένα.

Το πρώτο πράγμα που πρέπει να κάνετε είναι να εγκαταστήσετε το rdiff-backup και στον απομακρυσμένο υπολογιστή (ο κάθε υπολογιστής που θέλετε να κάνετε backup, συμπεριλαμβανομένου και του backup server, πρέπει να έχει εγκατεστημένο το rdiff-backup).

Για να κάνετε backup τον απομακρυσμένο υπολογιστή πάνω στον τοπικό υπολογιστή, τρέξτε στο τοπικό υπολογιστή (πχ 192.168.0.1): Προσέξτε την χρήση της διπλή άνω και κάτω τελείας :: 

# mkdir /media/sdb1/rdiff-backups/192.168.0.2/root
# rdiff-backup --exclude '/tmp/*' --exclude '/proc/*' --exclude '/sys/*' --exclude '/media/*/*' 192.168.0.2::/ /media/sdb1/rdiff-backups/192.168.0.2/root

Τώρα εάν θέλετε να επαναφέρετε έναν κατάλογο στον απομακρυσμένο υπολογιστή, ξεκινήστε την διαδικασία επαναφοράς είτε στον τοπικό υπολογιστή ή στον απομακρυσμένο υπολογιστή.

Αυτός είναι ο τρόπος με τον οποίο θα επαναφέρετε τον κατάλογο /usr/local/games σε ένα απομακρυσμένο υπολογιστή, ξεκινώντας την διαδικασία από τον απομακρυσμένο υπολογιστή:

# rdiff-backup -r now 192.168.0.1::/media/sdb1/rdiff-backups/192.168.0.1/root/usr/local/games /usr/local/games

Αυτός είναι ο τρόπος με τον οποίο θα επαναφέρετε τον κατάλογο /usr/local/games σε ένα απομακρυσμένο υπολογιστή, ξεκινώντας την διαδικασία από τον τοπικό υπολογιστή:

# rdiff-backup -r now /media/sdb1/rdiff-backups/192.168.0.1/root/usr/local/games 192.168.0.2::/usr/local/games

Η ίδια σύνταξη χρησιμοποιείται όταν επαναφέρετε την root κατάτμηση σας από ένα live cd (ο απομακρυσμένος υπολογιστής έχει εκκινηθεί μέσω live cd. Δείτε πιο πάνω).

Αυτοματοποίηση των backups:

Εάν κάνετε backup άλλους υπολογιστές πάνω στον τοπικό υπολογιστή σας, το πρώτο πράγμα που πρέπει να κάνετε είναι να ενεργοποιήσετε συνδέσεις (login) χωρίς password με την χρήση ssh κλειδιών. Σημειώστε ότι αυτό μιλάει για συνδέσεις ssh σαν υπέρ-χρήστης χωρίς password. Αυτό μπορεί να περιοριστεί έτσι ώστε να εκτελούνται μόνο rdiff-backup εντολές αλλά αυτό είναι εκτός των σκοπών του παρόντος θέματος. Παρακαλώ αναφερθείτε στο to SSH Ρύθμιση . Θα υποθέσουμε ότι υπάρχει απόλυτη εμπιστοσύνη και θα ρυθμίσουμε την απλούστερη μέθοδο για να επιτύχουμε συνδέσεις χωρίς password.

Από τον τοπικό υπολογιστή, κάντε τα ακόλουθα:

# [ -f /root/.ssh/id_rsa ] || ssh-keygen -t rsa -f /root/.ssh/id_rsa

Και πατήστε δυο φορές το enter για κενά (blank) passwords. Μετά κάντε:

# cat /root/.ssh/id_rsa.pub | ssh 192.168.0.2 'mkdir -p /root/.ssh;\
> cat - >> /root/.ssh/authorized_keys2'

Θα σας ζητηθεί το password υπέρ-χρήστη (root).

Τώρα μπορείτε να συνδεθείτε στον απομακρυσμένο υπολογιστή σαν υπέρ-χρήστης χωρίς να χρειάζεται να πληκτρολογήσετε password και το rdiff-backup μπορεί να αυτοματοποιηθεί.

Μετά δημιουργήστε ένα bash script το οποίο περιέχει όλες τις rdiff-backup εντολές. Το bash script μας μπορεί να μοιάζει κάπως έτσι:

#!/bin/bash
RDIFF=/usr/bin/rdiff-backup
echo
echo "=======Backing up 192.168.0.1 root======="
${RDIFF} --ssh-no-compression --exclude '/tmp/*' --exclude '/proc/*' --exclude '/sys/*' --exclude '/media/*/*' / /media/sdb1/rdiff-backups/192.168.0.1/root
echo "(and purge increments older than 1 month)"
${RDIFF} --remove-older-than 1M --force /media/sdb1/rdiff-backups/192.168.0.1/root
echo
echo "=======Backing up 192.168.0.1 mount sda5======="
${RDIFF} --ssh-no-compression --exclude /media/sda5/myjunk /media/sda5 /media/sdb1/rdiff-backups/192.168.0.1/sda5
echo "(and purge increments older than 1 months)"
${RDIFF} --remove-older-than 1M --force /media/sdb1/rdiff-backups/192.168.0.1/sda5
echo
echo "=======Backing up 192.168.0.2 root======="
${RDIFF} --ssh-no-compression --exclude '/tmp/*' --exclude '/proc/*' --exclude '/sys/*' --exclude '/media/*/*' --exclude '/mnt/*/*' 192.168.0.2::/media/sdb1/rdiff-backups/192.168.0.2/root
echo "(and purge increments older than 1 months)"
${RDIFF} --remove-older-than 1M --force /media/sdb1/rdiff-backups/192.168.0.2/root

Τώρα μπορείτε να ονομάσετε αυτό το bash script "myrdiff-backups.bash" και να το βάλετε μέσα στο /usr/local/bin πάνω στον τοπικό (local) σας υπολογιστή (backup server) και να το κάνετε εκτελέσιμο (executable). Τρέξτε το bash script και σιγουρευτείτε ότι δουλεύει.

Τέλος μπορείτε να βάλετε το cron να το καλεί κάθε νύχτα ώρα 8μμ. Η πιο κάτω γραμμή μέσα στο crontab του διαχειριστή (root) θα κάνει το κόλπο. Δώστε:

# crontab -e
και προσθέστε την πιο κάτω γραμμή
0 20 * * * /usr/local/bin/myrdiff-backups.bash
