konq-plugins - plugins για τον Konqueror.

Το πακέτο αυτό περιέχει διάφορα χρήσιμα plugins για τον Konqueror, τον διαχειριστή αρχείων, περιηγητή διαδικτύου (web browser) και document viewer του KDE. Πολλά από τα plugins θα εμφανιστούν στο μενού Εργαλεία (Tools) του Konqueror.

apt-get install konq-plugins

Τα σημαντικότερα για web browsing περιλαμβάνουν μεταφραστή σελίδας, αυτόματη ανανέωση σελίδας (auto-refreshing), αρχειοθέτηση σελίδων, δομική ανάλυση HTML και CSS, μπάρα αναζήτησης, πλαϊνή μπάρα news ticker, γρήγορη πρόσβαση σε συνηθισμένες επιλογές, bookmarklets, ένα crash monitor, ένας δείκτης διαθεσιμότητας microformat, μια del.icio.us bookmarks πλαϊνή μπάρα και ενσωμάτωση με τον aKregator RSS feed reader.

Τα σημαντικότερα για directory browsing (περιήγηση καταλόγων) περιέχουν φίλτρα καταλόγων, δημιουργία image gallery, συμπίεση και αποσυμπίεση αρχείων, γρήγορη αντιγραφή/μεταφορά (copy/move), μια πλαϊνή μπάρα για media player, μια πλαϊνή μπάρα με πληροφορίες για το αρχείο, ένας βοηθητικός κατάλογος για media, γραφικό πρόγραμμα που δείχνει το μέγεθος των καταλόγων (disk usage viewer) και μετατροπές, διαφοροποιήσεις εικόνων.

KDE Deskop Search - (Nepomuk and Strigi)

KDE's Nepomuk semantic desktop search is not enabled on an aptosid-kde.iso. To enable it, do the following:

Install the virtuoso-minimal and strigi-client packages:

apt-get install virtuoso-minimal strigi-client

Navigate to System Settings > System Administration > Startup and Shutdown >Service Manager, put a check mark in the box for Nepomuk Search Module, and then click on Apply.

Navigate to System Settings > Workspace Appearance and Behavior > Desktop Search > Basic Settings, and put check marks in the boxes for Enable Nepomuk Semantic Desktop and Enable Strigi Desktop File Indexer, and then click the Apply. Note that the amount of memory allocated to the Nepomuk database can be adjusted in the Advanced Settings tab.

The KDE session must be restarted for the indexing service to become effective. Expect the first iteration of indexing to take a long time. More information about Nepomuk .

Δεν μπορώ να εισέλθω στο KDE

To περιεχόμενο φακέλου /tmp λογικά καθαρίζεται με κάθε εκκίνηση, έτσι μερικοί φάκελοι και sockets που χρειάζονται από τον X-Server σβήνονται επίσης.

Λογικά, κατα τη διάρκεια της εκκίνησης, το script x11-common για τον X-Org τα επαναδημιουργεί.

Πιθανώς αυτά τα scripts δεν καλούνται κατά την εκκίνηση. Δημιουργήστε ξανά τους συνδέσμους (links) με την εντολή :

# X-ORG: # dpkg-reconfigure x11-common

Το KDE χρειάζεται 5% της κατάτμησης στην οποία βρίσκεται ο φάκελος /tmp για τα μη μόνιμα (temporary) αρχεία που δημιουργούνται κατά την σύνδεση (login). Εάν τρέχετε με μια κατάτμηση που είναι 95% γεμάτη δεν θα μπορέσετε να συνδεθείτε με το KDE και θα 'πέσετε' μέσα στο tty.

Το ίδιο ισχυεί και όταν το kdm 'κάνει' κύκλους (looping) αλλά δεν σας αφήνει να συνδεθείτε. Μια λύση είναι να συνδεθείτε μέσα στο tty για να μπορέσετε να σβήσετε ή/και να καθαρίσετε το σύστημα από εφαρμογές και αρχεία που δεν χρειάζονται πλέον.

Εναλλακτικά μπορείτε να χρησιμοποιήσετε ένα X διαχειριστή παραθύρων (window manager) που δεν χρειάζεται τόσο πολλή χώρο από το σύστημα για να λειτουργήσει (για παράδειγμα ο fluxbox υπάρχει ήδη σε μια εγκατάσταση aptosid) ή να κάνετε chroot χρησιμοποιώντας ένα aptosid live-CD/DVD για να καθαρίσετε την κατάτμηση για να σας επιτραπεί να εκκινήσετε το KDE.

85% είναι το συνιστώμενο απόλυτο μέγιστο για μια κατάτμηση στην οποία χρειάζεται πρόσβαση το KDE για τον /tmp φάκελο της.(15% άδεια (free)).

Display Managers - dm

Εγκατάσταση άλλων επιφάνειων εργασείας μαζί με το προεγκατεστημένο περιβάλλον εργασίας:

Όταν θέλετε να εγκαταστήσετε ακόμη ένα περιβάλλον εργασίας μαζί με αυτό που έχετε ήδη εγκατεστημένο, (για παράδειγμα έχετε εγκαταστήσει το aptosid-kde.iso και τώρα θέλετε να εγκαταστήσετε το Xfce ή το LXDE περιβάλλον εργασίας), το πιο πιθανών είναι να εγκατασταθεί και ένας display manager (dm) μαζί του ή ίσως έχετε εγκαταστήσει εσείς ένα 2ο dm (gdm, slim ή κάποιο άλλο πακέτο dm).

The problem with this is you will end up with the default Debian runlevel configuration with the consequence that you will need to stop X manually in runlevel 3 before commencing a dist-upgrade.

Η λύση είναι:

apt-get update
apt-get install --reinstall distro-defaults
update-rc.d <dm> remove
update-rc.d <dm> defaults

Παραδείγματα για update-rc.d <dm> defaults και προσέξτε την τελεία . :

update-rc.d kdm start 24 5 . stop 01 0 1 2 3 4 6 .

update-rc.d gdm start 24 5 . stop 01 0 1 2 3 4 6 .

update-rc.d slim start 01 5 . stop 01 0 1 2 3 4 6 .

Η επιφάνεια εργασίας δεν ανταποκρίνεται

Σε μια τέτοια περίπτωση δεν χρειάζεστε πάντα το κουμπί του reset. Aυτό θα μπορούσε να καταστρέψει το σύστημα αρχειων ή να οδηγήσει σε απώλεια δεδομένων. Σε κάθε περίπτωση το σύστημα αρχείων δεν θα είναι καθαρό μετά από hard reset (filesystem not clean)

Πρώτα δοκιμάστε να μεταφερθείτε σε κονσόλα με alt-ctl-F1 ή να επανεκκινήσετε τον X-server με alt-ctrl-backspace, (Αν καμία από τις δύο αυτές επιλογές δεν λειτουργήσει, υπάρχει ακόμα ελπίδα):

Το SYSRQ-πλήκτρο (το print-πλήκτρο, στην πάνω δεξιά μεριά του πληκτρολογίου) μπορεί να βοηθήσει για μια καθαρή επανεκκίνηση ενός μη ανταποκρινόμενου συστήματος.

Η παρακάτω ακολουθία από συνδιασμούς πλήκτρων είναι πιθανές:
* alt+sysrq+r (θα πρέπει να επαναφέρει τον έλεγχο του πληκτρολογίου)
* alt+sysrq+s (Εκδίδει ένα sync (issues a sync))
* alt+sysrq+e (στέλνει την εντολή term σε όλες τις διεργασίες εκτός από το init)
* alt+sysrq+i (στέλνει την εντολή kill σε όλες τις διεργασίες εκτός από το init)
* alt+sysrq+u (τα συστήματα αρχείων προσαρτούνται μόνο για ανάγνωση. Aποτρέπει το fsck κατά την επανεκκίνηση)
* alt+sysrq+b (επανεκκινεί το σύστημα. Χωρίς τα προηγούμενα βήματα θα είναι hard reset).

Eίναι καλύτερο να αφήσετε σε κάθε βήμα μερικά δευτερόλεπτα για να ολοκληρωθεί. Π.χ. ο τερματισμός όλων των διεργασιών χρειάζεται κάποιο χρόνο. Tα γράμματα τα οποία χρειάζονται μπορούν έυκολα να απομνημονευτούν με την βοήθεια της παρακάτω φράσης:
"Reboot System Even If Utterly Broken"

Another way to remember it is:
"Raising Skinny Elephants Is Utterly Boring"
