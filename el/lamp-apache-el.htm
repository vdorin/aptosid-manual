aptosid LAMP Stack

Το ακρώνυμο LAMP αναφέρεται σε ένα σετ από ελεύθερου λογισμικού εφαρμογές που χρησιμοποιούνται συνήθως σε συνεργασία για το τρέξιμο δυναμικών ιστοσελίδων ή διακομιστών (servers)
Linux, το λειτουργικό σύστημα
Apache, ο διακομιστής ιστοσελίδων (web server)
MySQL, το σύστημα διαχείρισης βάσεων δεδομένων (ή ο διακομιστής βάσεων δεδομένων (database server))
Perl, PHP, ή/και Python, γλώσσες για scripting

ΠΡΟΕΙΔΟΠΟΙΗΣΗ: Ποτέ μην χρησιμοποιείτε τον ηλεκτρονικό σας υπολογιστή που έχετε για τις καθημερινές δουλειές σας να παίζει τον ρόλο διαδικτυακού διακομιστή ιστοσελίδων! Χρησιμοποιήστε ένα ειδικό υπολογιστή για διαδικτυακό διακομιστή ιστοσελίδων και μην τον αφήσετε να κάνει τίποτε άλλο!

Χρήσεις διακομιστή:
a) σαν τοπικός διακομιστής για τεστ (local test server) για σχεδιαστές ιστοσελίδων χωρίς σύνδεση στο διαδίκτυο, που είναι και ο σκοπός αυτού του θέματος,
b) σαν ιδιωτικός διακομιστής "ντουλάπας" ("closet" server) με σύνδεση στο δίκτυο,
c) σαν ιδιωτικός διακομιστής ιστοσελίδων με πλήρη διάδοση στο διαδίκτυο
d) σαν εμπορικός διακομιστής ιστοσελίδων, που είναι πέραν του σκοπού αυτού του εγχειριδίου

Ελάχιστες Απαιτήσεις

Πρέπει να έχετε ελεύθερα τουλάχιστον 256MB RAM. Οτιδήποτε λιγότερο από αυτό το ελάχιστο θα δημιουργήσει πολλά προβλήματα γιατί το τρέξιμο ενός διακομιστή με mysql χρειάζεται πολλή RAM για να τρέξει σωστά. Η Mysql θα βγάλει σαν μήνυμα λάθους "cannot connect to mysql.sock" εάν δεν έχει αρκετή μνήμη ο διακομιστής σας.

Για να ελέγξετε αν η php είναι εγκατεστημένη και δουλεύει σωστά, απλά δημιουργήστε ένα test.php αρχείο μέσα στο /var/www φάκελο σας με phpinfo() εντολή ακριβώς όπως σας δείχνετε από κάτω.

Γράψτε στον περιηγητή (browser) δικτύου σας:

http://localhost/test.php
ή

Αυτό θα πρέπει να δείξει όλες τις php ρυθμίσεις (configuration) και προεπιλεγμένες ρυθμίσεις (default settings).

Μπορείτε να επεξεργαστείτε απαραίτητες τιμές ή να ορίσετε εικονικά domains χρησιμοποιώντας το αρχείο ρυθμίσεων του apache.

Αν θέλετε να δοκιμάσετε την εγκατάσταση σας πηγαίνετε στον περιηγητή δικτύου σας και γράψτε τα ακόλουθα

http://η_ip_διεύθυνση_σας/apache2-default/

Αν η εγκατάσταση έγινε με επιτυχία και σωστά, θα δείτε ένα μήνυμα καλώς ορίσματος.

mkdir /home/ο_εαυτός_μου/www

Εκτελώντας τις παραπάνω εντολές, μπορείτε να επεξεργαστείτε την ιστοσελίδα σας μέσα στο home σαν απλός χρήστης.

Βασική Προστασία των Αρχείων του Διακομιστή

Μια πτυχή του Apache η οποία παρερμηνεύεται περιστασιακά είναι το χαρακτηριστικό (feature) της βασικής πρόσβασης (default access). Αυτό σημαίνει, αν δεν ληφθούν μέτρα για να αλλαχθεί αυτό, ότι αν ο διακομιστής μπορεί να βρει το μονοπάτι ενός αρχείου μέσω συνηθισμένων κανόνων χαρτογράφησης (mapping rules) URL, τότε μπορεί να το δώσει (serve) στους πελάτες (clients).

Για παράδειγμα δείτε την παρακάτω περίπτωση:

 1. # cd /; ln -s / public_html
 2. Accessing http://localhost/~root/

Αυτό θα επέτρεπε στους πελάτες να δουν όλο το σύστημα αρχείων σας! Για να το αποφύγετε αυτό, προσθέστε τα παρακάτω στις ρυθμίσεις του διακομιστή σας:

<Directory />
   Order Deny,Allow
   Deny from all
</Directory>

Αυτό θα απαγορέψει την βασική πρόσβαση σε σημεία του συστήματος αρχείων. Προσθέστε τις κατάλληλες <Directory> γραμμές για να επιτρέψετε την πρόσβαση μόνο στους χώρους που θέλετε. Για παράδειγμα ,

<Directory /usr/users/*/public_html>
    Order Deny,Allow
    Allow from all
</Directory>
<Directory /usr/local/httpd>
    Order Deny,Allow
    Allow from all
 </Directory>

Δώστε ιδιαίτερη προσοχή στις αλληλεπιδράσεις (interactions) των <Location> και των <Directory> οδηγιών; Για παράδειγμα αν το <Directory /> δεν επιτρέπει την πρόσβαση, μια <Location /> οδηγία μπορεί να το ανατρέψει.

Επίσης να είστε επιφυλακτική με την UserDir οδηγία. Ρυθμίζοντας το σε κάτι όπως "./" θα έχει το ίδιο αποτέλεσμα, για τον root, όπως το πρώτο παράδειγμα πιο πάνω. Εάν χρησιμοποιείτε Apache 1.3 ή νεότερο, σας συνιστούμε να συμπεριλάβετε την πιο κάτω γραμμή στα αρχεία ρύθμισης του διακομιστή σας:

UserDir disabled root

Τρέξτε το script “a2enmod ssl” πχ

 # a2enmod ssl

Αυτό θα δημιουργήσει αυτόματα ένα συμβολικό σύνδεσμο μεταξύ mods- available και mods – enabled

Δημιουργήστε ένα αντίγραφο του αρχείου '/etc/apache2/sites-available/default' μέσα στον φάκελο /etc/apache2/sites-available/ - ονομάστε το 'ssl'

# cp /etc/apache2/sites-available/default /etc/apache2/sites-available/ssl

Δημιουργήστε ένα συμβολικό σύνδεσμο για το νέο αρχείο ρύθμισης για να το χρησιμοποιήσετε

#  ln -s /etc/apache2/sites-available/ssl /etc/apache2/sites-enabled/
(ή)
#a2ensite ssl

Εάν θέλετε να αλλάξετε βασικές ρυθμίσεις κάντε το μέσα στο /etc/apache2/apache2.conf και αν θέλετε να κάνετε αλλαγές στο βασικό document root κάντε τις μέσα στο αρχείο /etc/apache2/sites-available/default και επανεκκινήστε τον διακομιστή apache.

Για να επανεκκινήσετε τον διακομιστή Apache, χρησιμοποιείστε την πιο κάτω εντολή

#service apache2 restart

Τώρα πρέπει να αλλάξουμε το port μέσα στο /etc/apache2/ports.conf . Εκ προεπιλογής θα "ακούει" το port 80 και επειδή τώρα κάνουμε εγκατάσταση με SSL, πρέπει να το αλλάξουμε για να "ακούει" το port 443

 Listen 443

Επεξεργαστείτε το /etc/apache2/sites-available/ssl (ή οποιοδήποτε όνομα δώσατε στο αρχείο ρύθμισης του ssl σας) και αλλάξτε το port 80 σε port 443.

Προσθέστε τις πιο κάτω γραμμές στο αρχείο /etc/apache2/apache2.conf

  SSLEngine On

  SSLCertificateFile /etc/apache2/ssl/apache.pem

Επεξεργαστείτε το SSLCertificateFile /etc/apache2/ssl/apache.pem και καταχωρίστε τις τοποθεσίες του πιστοποιητικού αρχείου (certificate file) και του πιστοποιητικού κλειδιού αρχείου (certificate key file). Από κάτω είναι ένα παράδειγμα

SSLCertificateFile /etc/apache2/ssl/online.test.net.crt
SSLCertificateKeyFile /etc/apache2/ssl/online.test.net.key

Για να θέσετε το ServerSignature στο off (κλειστό), επεξεργαστείτε το αρχείο /etc/apache2/apache2.conf και προσθέστε αυτές τις δύο γραμμές

 ServerSignature Off
 ServerTokens ProductOnly

Αν θέλετε να επιτρέπετε διαφόρων τύπων αρχεία index, κοιτάξτε για την πιο κάτω γραμμή μέσα στο /etc/apache2/apache2.conf αρχείο

 DirectoryIndex index.html index.cgi index.pl index.php index.xhtml index.shtml

Επανεκκινήστε τον διακομιστή apache

 service apache2 restart

Τώρα θα πρέπει να έχετε ένα διακομιστή για δοκιμές. Θέλετε να ενωθείτε στο διαδίκτυο με αυτόν, ΜΗΝ ΤΟ ΚΑΝΕΤΕ! ... Χρησιμοποιείστε ένα άλλο υπολογιστή που είναι αφιερωμένος στο να είναι διακομιστής διαδικτύου
