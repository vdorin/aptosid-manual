Βασικός Οδηγός για το πως να Κάνετε το Ασύρματο (Wireless - WiFi) σας να δουλέψει

Λόγω της περιπλοκότητας του νόμου, το aptosid θα παρέχει μόνο dfsg-ελεύθερο λογισμικό, παρακαλώ κοιτάξτε αυτή τη διεύθυνση για επιπλέον πληροφορίες για πηγές μη ελεύθερου firmware

Για να κάνετε το wlan WiFi (ασύρματο) να δουλέψει, χρειάζεστε μια ενσύρματη σύνδεση για μερικά λεπτά για να κατεβάσετε το κατάλληλο firmware.

Εάν είναι αδύνατον να έχετε μια ενσύρματη σύνδεση, θα πρέπει να βάλετε το firmware πάνω σε μια αφαιρούμενη συσκευή (φλασακί π.χ.) και να το εγκαταστήσετε σαν υπέρ-χρήστης (root) από εκεί χρησιμοποιώντας:

#dpkg -i <firmware.deb>

Για να βρείτε το κατάλληλο firmware χωρίς να ξέρετε την μάρκα της ασύρματης κάρτας σας, μπορείτε να χρησιμοποιήσετε την εντολή:

#fw-detect

Αυτό θα σας δώσει τις ακόλουθες πληροφορίες:

#apt-get update
#apt-get install <όνομα του firmware>
#modprobe -r <όνομα του module>
#modprobe <όνομα του module>

Χρησιμοποιήστε την apt-get install γραμμή που σας έδωσε η εντολή fw-detect . Όταν τελειώσει αυτό, θα πρέπει να δώσετε μερικές εντολές στην κονσόλα πριν ρυθμίσετε την συσκευή.

Τώρα θα πρέπει να φορτώσετε την ενότητα (module) για να μπορέσετε να ρυθμίσετε την συσκευή σας.

Σαν υπέρ-χρήστης μέσα στην κονσόλα γράψτε:

modprobe -r <όνομα module>

modprobe <όνομα module>

Επειδή το <όνομα του module> χρησιμοποιεί τις πληροφορίες που το fw-detect σας έδωσε πριν, μια χρήσιμη λειτουργία της κονσόλας μπορεί να χρησιμοποιηθεί εδώ: bash completion (συμπλήρωση):
Αν απλά γράψετε τα πρώτα γράμματα του <ονόματος του module> και πατήσετε το TAB, συμπληρώνετε το όνομα του module (π.χ. modprobe ipw TAB ). Αυτό εμποδίζει επίσης την λάθος γραφή μιας λέξης.

Και οι δυο modprobe εντολές δεν έχουν κάποια έξοδο (output) αν είναι επιτυχής γιαυτό αν σας επιστραφεί η συνηθισμένη οθόνη, το module έχει φορτωθεί σωστά.

Μπορείτε να το ελέγξετε με:

#lsmod | grep <module>

Τώρα ξεκινήστε το Ceni από το K-Menu - Internet ή τρέξτε το από την κονσόλα σαν υπέρ-χρήστης με Mπαίνοντας στο Δίκτυο - Ceni. Η ασύρματη WiFi σας συσκευή θα πρέπει να φαίνεται και είναι τώρα έτοιμη να ρυθμιστεί.

Για να ρυθμίσετε ένα Wlan GUI δείτε WiFi - roaming WPA-GUI

Τρόποι λειτουργίας του wpasupplicant για το Debian

Λόγω της περιπλοκότητας του νόμου, το aptosid θα παρέχει μόνο dfsg-ελεύθερο λογισμικό, παρακαλώ κοιτάξτε αυτή τη διεύθυνση για επιπλέον πληροφορίες για πηγές μη ελεύθερου firmware

Το Debian wpasupplicant πακέτο παρέχει δυο (2) βολικούς τρόπους λειτουργίας οι οποίοι είναι στενά συνδεδεμένοι με τη βάση της δικτυακής υποδομής (ifupdown).

Τι καλύπτεται

1. Διευκρινίζοντας το wpa_supplicant driver backend

* Πίνακας υποστηριζόμενων οδηγών
* Συστάσεις Οδηγών

2. Mode #1: Managed Mode

* Παραδείγματα
* Πίνακας Συνηθισμένων Επιλογών
* Σημαντικές Σημειώσεις για το Managed Mode
* Πως λειτουργεί

3. Mode #2: Roaming (περιαγωγή) Mode

* wpa_supplicant.conf
* /etc/network/interfaces
* Έλεγχος του Roaming Daemon με wpa_action
* Ρύθμιση λεπτομερειών του Roaming Setup
* Το αρχείο ημερολόγιου (Logfile)
* Χρήση εξωτερικών script χαρτογράφησης (External Mapping Scripts) (π.χ. guessnet)
* /etc/network/interfaces με εξωτερική χαρτογράφηση

4. Καταπολέμιση προβλημάτων (troubleshooting)

* Κρυμμένα ssids

5. Ασφάλεια

* Ρύθμιση Δικαιωμάτων Αρχείου

1. Διευκρινίζοντας το wpa_supplicant driver backend

Εάν καμία από τις συσκευές δεν επιλέγει συγκεκριμένο οδηγό στο 'wpa-driver' τότε χρησιμοποιείται ο οδηγός 'wext'. Χρήστες πυρήνων linux τις σειράς 2.4 ή τις σειράς 2.6 (παλαιότερους του 2.6.14) θα χρειαστεί να επιλέξουν ένα τύπο wpa-driver.

Υποστηριζόμενοι Οδηγοί

Οδηγός Περιγραφή
wext Linux ασύρματη επέκταση (wireless extensions) (γενικό)

Συστάσεις Οδηγών

Οι Intel Pro Wireless κάρτες (ipw2100, ipw2200 και ipw3945) χρησιμοποιούν το 'wext' backend, εκτός και αν ο πυρήνας σας είναι παλαιότερος από τον 2.6.14

Το Madwifi υποστηρίζει και το 'wext' και το 'madwifi' driver (οδηγός) backends. Το 'wext' προτιμάται, αλλά το 'madwifi' μπορεί να δουλέψει καλύτερα σε ορισμένες περιπτώσεις.

Το Ndiswrapper ΔΕΝ ΥΠΟΣΤΗΡΙΖΕΙ ΠΛΕΟΝ (έκδοση 1.16 και νεότερες) το 'ndiswrapper' driver backend. Γιαυτό πρέπει να χρησιμοποιήσετε το 'wext' εκτός και αν χρησιμοποιείτε μια απαρχαιωμένη έκδοση του ndiswrapper.

Ρυθμίστε τον τύπο του οδηγού, μέσα στην interfaces(5) παράγραφο, για την συσκευή σας με την 'wpa-driver' επιλογή. Για παράδειγμα:

iface eth0 inet dhcp
	wpa-driver wext
	. . . . . άλλες επιλογές

2. Mode #1: Managed Mode

Αυτός ο τρόπος λειτουργίας δίνει την δυνατότητα δημιουργίας μιας σύνδεσης μέσω του wpa_supplicant με ένα γνωστό δίκτυο. Λειτουργεί με τρόπο παρόμοιο της λειτουργίας του wireless-tools (ασύρματα εργαλεία) πακέτου. Το κάθε στοιχείο που χρειάζεται για να δημιουργηθεί η σύνδεση μέσω wpa_supplicant έχει σαν πρόθεμα 'wpa-' και ακολουθείται από την τιμή που θα χρησιμοποιηθεί γιαυτό το στοιχείο.

Παραδείγματα ενός wpa.conf αρχείου του πρώτου τρόπου λειτουργίας (Mode #1) - Παράδειγμα 1.
Example of a Mode #1 wpa.conf file - Example 1.
NOTE: the 'wpa-psk' value is only valid if:

   1. It is a plaintext (ascii) string between 8 and 63 characters in length
   2. It is a hexadecimal string of 64 characters

# Connect to access point of ssid 'NETBEER' with an encryption type of
# WPA-PSK/WPA2-PSK. It assumes the driver will use the 'wext' driver backend
# of wpa_supplicant because no wpa-driver option has been specified.
# The passphrase is given as a ASCII (plaintext) string. DHCP is used to
# obtain a network address.
#
iface wlan0 inet dhcp

        wpa-ssid NETBEER
        # plaintext passphrase
        wpa-psk PlainTextSecret

# Connect to access point of ssid 'homezone' with an encryption type of
# WPA-PSK/WPA2-PSK, using the 'wext' driver backend of wpa_supplicant.
# The psk is given as an encoded hexadecimal string. DHCP is used to obtain
# a network address.
#
iface wlan0 inet dhcp
wpa-driver wext
wpa-ssid homezone
# hexadecimal psk is encoded from a plaintext passphrase
wpa-psk 000102030405060708090a0b0c0d0e0f101112131415161718191a1b1c1d1e1f

# Connect to access point of ssid 'HotSpot1' and bssid of '00:1a:2b:3c:4d:5e'
# with an encryption type of WPA-PSK/WPA2-PSK, using the the 'madwifi' driver
# backend of wpa_supplicant. The passphrase is given as a plaintext string.
# A static network address assignment is used.
#
iface ath0 inet static

        wpa-driver madwifi
        wpa-ssid HotSpot1
        wpa-bssid 00:1a:2b:3c:4d:5e
        # plaintext passphrase
        wpa-psk madhotspot
        wpa-key-mgmt WPA-PSK
        wpa-pairwise TKIP CCMP
        wpa-group TKIP CCMP
        wpa-proto WPA RSN
        # static ip settings
        address 192.168.0.100
        netmask 255.255.255.0
        network 192.168.0.0
        broadcast 192.168.0.255
        gateway 192.168.0.1

# User supplied wpa_supplicant.conf is used for eth1. All network information
# is contained within the user supplied wpa_supplicant.conf. No wpa-driver type
# is specified, so wext is used. DHCP is used to obtain a network address.
#
iface eth1 inet dhcp

wpa-conf /path/to/wpa_supplicant.conf

Πίνακας Συνηθισμένων Επιλογών

Μια σύντομη περιγραφή των συνηθισμένων 'wpa-' επιλογών που μπορούν να χρησιμοποιηθούν στο /etc/network/interfaces για μια ασύρματη συσκευή. Δέστε το 'Σημαντικές Σημειώσεις για το Managed Mode' κομμάτι για πληροφορίες για επιτρεπόμενες και μη 'wpa-' τιμές.

ΣΗΜΕΙΩΣΗ: Όλες οι τιμές είναι CASE SeNsItVe (υπάρχει διαφορά ανάμεσα σε κεφαλαία γράμματα και μικρά)

Στοιχείο	Παράδειγμα Τιμής	Περιγραφή
========        ================        =========
wpa-ssid        plaintextstring         ρυθμίζει το ssid του δικτύου σας
wpa-bssid       00:1a:2b:3c:4d:5e       το bssid του AP
wpa-psk         0123456789......        το preshared wpa κλειδί σας. Χρησιμοποιήστε
                                        το wpa_passphrase(8) για να δημιουργήσετε το psk
                                        από ένα passphrase και ένα ssid
wpa-key-mgmt    NONE, WPA-PSK, WPA-EAP, λίστα αποδεχόμενων επικυρωμένων κλειδιών
                IEEE8021X               πρωτοκόλλων διαχείρισης
wpa-group       CCMP, TKIP, WEP104,     λίστα αποδεχόμενων αλγορίθμων κρυπτογράφησης (group ciphers)
                WEP40			για WPA
wpa-pairwise    CCMP, TKIP, NONE        λίστα αποδεχόμενων αλγορίθμων κρυπτογράφησης (pairwise ciphers)
                                        για WPA
wpa-auth-alg    OPEN, SHARED, LEAP      λίστα επιτρεπόμενων IEEE 802.11
                                        αλγορίθμων επικύρωσης
wpa-proto       WPA, RSN                λίστα αποδεχόμενων πρωτοκόλλων
wpa-identity    myplaintextname         όνομα χρήστη που δόθηκε από τον διαχειριστή
                                        (EAP authentication)
wpa-password    myplaintextpassword     το password σας (EAP authentication)
wpa-scan-ssid   0 ή 1                  	αλλάζει την ssid σάρωση με συγκεκριμένα
                                        Probe Request frames
wpa-ap-scan     0 ή 1 ή 2             	ρυθμίζει την λογική σάρωσης του
                                        wpa_supplicant

Η πλήρης λειτουργικότητα του wpa_cli (8) θα πρέπει να εφαρμοστεί (implemented). Οτιδήποτε λείπει θεωρείται λάθος (bug) και θα πρέπει να αναφέρεται ως τέτοιο. Patches είναι πάντα ευπρόσδεκτα.

Σημαντικές Σημειώσεις για το Managed Mode

Σχεδόν όλες οι 'wpa-' επιλογές απαιτούν να έχετε τουλάχιστον δώσει ένα ssid . Μόνο πολύ λίγες επιλογές έχουν καθολική επίδραση (global effect). Αυτές είναι: 'wpa-ap-scan' και 'wpa-preauthenticate'.

Οποιαδήποτε 'wpa-' επιλογή δοσμένη για μια συσκευή μέσα στο interfaces(5) αρχείο είναι αρκετή να θέσει σε λειτουργία το wpa_supplicant daemon.

Το wpasupplicant ifupdown script κάνει υποθέσεις για τον 'τύπο' εισροών (input) που είναι έγκυρες για κάθε επιλογή. Για παράδειγμα υποθέτει ότι μια εισροή είναι απλό κείμενο (plaintext) και την βάζει σε εισαγωγικά πριν την διαβιβάσει στο wpa_cli, το οποίο προσθέτει την εισροή στο δικτυακό μπλοκ που δημιουργείται μέσω του wpa_supplicant ctrl_interface socket.

Το τρέξιμο του ifup χειροκίνητα με την '--verbose' επιλογή θα αποκαλύψει όλες τις εντολές που χρησιμοποιούνται για να δημιουργηθεί το δικτυακό μπλοκ μέσω του wpa_cli. Εάν μια τιμή που χρησιμοποιήσατε για οποιαδήποτε wpa-* επιλογή μέσα στο /etc/network/interfaces είναι μέσα σε διπλά εισαγωγικά, τότε έχει γίνει η υπόθεση είναι τύπος εισροής "plaintext" ή "ascii".

Για ορισμένες εισροές γίνεται η υπόθεση ότι είναι hexadecimal (δεκαεξαδικός) string (πχ. wpa-wep-key*). Ο 'τύπος' της τιμής της wpa-psk επιλογής όμως, προσδιορίζεται μέσω ενός απλού ελέγχου για περισσότερους από ένα μη δεκαεξαδικούς χαρακτήρες.

Πως λειτουργεί

Όπως αναφέρθηκε νωρίτερα, κάθε wpa_supplicant συγκεριμένο αντικείμενο έχει το πρόθεμα 'wpa-'. Κάθε στοιχείο σχετίζεται με μια ιδιότητα του wpa_supplicant που περιγράφεται στις wpa_supplicant.conf(5), wpa_supplicant(8) και wpa_cli(8) manpages (σελίδες εγχειριδίου).

Το supplicant τρέχει χωρίς να είναι καθόλου προ-ρυθμισμένο και το wpa_cli δημιουργεί μια ρύθμιση δικτύου από τις εισροές (input) που παρέχονται από τις 'wpa-*' γραμμές. Αρχικά το wpa_supplicant/wpa_cli δεν ορίζει απευθείας τις ιδιότητες της συσκευής (όπως όταν ορίζουμε ένα essid με το iwconfig, για παράδειγμα), αλλά ενημερώνει την συσκευή για το πιο access point (σημείο πρόσβασης) είναι κατάλληλο για να συνεργαστεί μαζί του. Όταν η συσκευή σαρώσει την περιοχή και βρει ότι το κατάλληλο access point είναι διαθέσιμο για χρήση, ορίζονται οι ιδιότητες.

Το script που κάνει όλη την δουλειά βρίσκεται εδώ:

/etc/wpa_supplicant/ifupdown.sh
/etc/wpa_supplicant/functions.sh

Το ifupdown.sh εκτελείται από το run-parts, το οποίο με την σειρά του καλείται από το ifupdown κατά την διάρκεια των 'pre-up', 'pre-down' και 'post-down' φάσεων.

Στην 'pre-up' φάση, ένα wpa_supplicant daemon εκτελείτε, ακολουθούμενο από μια σειρά από wpa_cli εντολές που ρυθμίζουν το δίκτυο αναλόγως του ποιες 'wpa-' επιλογές χρησιμοποιήθηκαν στο /etc/network/interfaces για την συσκευή.

Εάν χρησιμοποιείται το wpa-roam , ένας wpa_cli daemon εκτελείται στην 'post-up' φάση.

Στην 'pre-down' φάση, ο wpa_cli daemon τερματίζεται (kill) εάν υπάρχει.

Στην 'post-down' φάση, ο wpa_supplicant daemon τερματίζεται (kill).

3. Mode #2: Roaming Mode

Ένας αυτόνομος, απλοϊκός μηχανισμός περιαγωγής (roaming) παρέχεται από αυτό το πακέτο. Είναι στην μορφή ενός wpa_cli action script, /sbin/wpa_action, και αναλαμβάνει τον έλεγχο του ifupdown μόλις ενεργοποιηθεί. Η wpa_action(8) manpage (σελίδα εγχειριδίου) περιγράφει τις τεχνικές του λεπτομέρειες σε μεγάλο βάθος.

Για να ενεργοποιήσετε μια διεπαφή περιαγωγής (roaming interface), αλλάξτε την παράγραφο του παραδείγματος στο interfaces(5):

iface eth1 inet manual
	wpa-driver wext
        wpa-roam /path/to/wpa_supplicant.conf

Δύο daemons αναπαράγονται (spawned) από το πιο πάνω παράδειγμα, ο wpa_supplicant και ο wpa_cli. Απαιτείται να παρέχεται ένα wpa_supplicant.conf (αρχείο ρυθμίσεων). Ένα καλό σημείο εκκίνησης παρέχεται από ένα παράδειγμα αρχείου ρυθμίσεων:

 # copy the template to /etc/wpa_supplicant/
        cp /usr/share/doc/wpasupplicant/examples/wpa-roam.conf \
                /etc/wpa_supplicant/wpa_supplicant.conf
        # allow only root to read and write to file
        chmod 0600 /etc/wpa_supplicant/wpa_supplicant.conf

NOTE: it is critical that the used wpa_supplicant.conf defines the location of
      the 'ctrl_interface' so that a communication socket is created for the
      wpa_cli (wpa-roam daemon) to attach. The mentioned example configuration,
      /usr/share/doc/wpasupplicant/examples/wpa-roam.conf, has been set to a
      sane default.

Απαιτείται να επεξεργαστείτε το αρχείο ρυθμίσεων και να προσθέσετε δικτυακά μπλοκ για όλα τα γνωστά δίκτυα. Εάν δεν καταλαβαίνετε τι σημαίνει αυτό, αρχίστε να διαβάζετε την wpa_supplicant.conf(5) manpage τώρα.

Για κάθε δίκτυο μπορείτε να καθορίσετε μια ειδική επιλογή 'id_str'. Το 'id_str' θα πρέπει να είναι ένα απλό text string. Αυτό το string αποτελεί τη βάση για δικτυακά προφίλ. Συσχετίζεται με μια λογική διεπαφή (logical interface) καθορισμένη μέσα στο interfaces(5) αρχείο. Εάν δεν είναι δοσμένο ένα 'id_str' για κάποιο δίκτυο, το wpa_action υποθέτει ότι θα χρησιμοποιήσει την λογική διεπαφή προεπιλογής ('default' logical interface) σαν εφεδρική διεπαφή. Η εφεδρική διεπαφή μπορεί να επιλεχτεί μέσω της 'wpa-default-iface' επιλογής.

Τί σημαίνουν όμως όλα αυτά; Ας δούμε ένα μικρό παράδειγμα παρμένο από τη wpa_action(8) manpage.

wpa_supplicant.conf παράδειγμα
wpa_supplicant.conf example:
network={
        ssid="foo"
        key_mgmt=NONE
        # this id_str will notify /sbin/wpa_action to 'ifup uni'
        id_str="uni"
}

network={
        ssid="bar"
        psk=123456789...
        # this id_str will notify /sbin/wpa_action to 'ifup home_static'
        id_str="home_static"
}

network={
        ssid=""
        key_mgmt=NONE
        # no 'id_str' parameter is given, /sbin/wpa_action will 'ifup default'
}
/etc/network/interfaces παράδειγμα
/etc/network/interfaces example:
# the roaming interface MUST use the manual inet method
# 'allow-hotplug' or 'auto' ensures the daemon starts automatically
allow-hotplug eth1
iface eth1 inet manual

        wpa-driver wext
        wpa-roam /etc/wpa_supplicant/wpa_supplicant.conf


# no id_str, 'default' is used as the fallback mapping target
iface default inet dhcp


# id_str="uni"
iface uni inet dhcp

# id_str="home_static"
iface home_static inet static

        address 192.168.0.20
        netmask 255.255.255.0
        network 192.168.0.0

        broadcast 192.168.0.255
        gateway 192.168.0.1

Μια λογική διεπαφή 'ξεκινάει' μέσω του ifup και τερματίζεται μέσω του ifdown καθώς το wpa_supplicant συνεργάζεται και σταματάει την συνεργασία του με το δίκτυο που του ανατέθηκε από την 'id_str' επιλογή που χρησιμοποιήθηκε μέσα στο the wpa_supplicant.conf αρχείο ρυθμίσεων.

Ένα αρχείο ημερολογίου (log) των πράξεων του /sbin/wpa_action δημιουργείται στο /var/log/wpa_action.log , παρακαλώ επισυνάπτετε και το log όταν αναφέρεται προβλήματα.

Αλληλεπίδραση με το wpa_supplicant μέσω του wpa_cli και του wpa_gui

Μέλη της ομάδας "netdev" μπορούν να αλληλεπιδράσουν με την wpa_supplicant διεργασία εάν η ρυθμίσεις περιαγωγής του παραδείγματος έχουν χρησιμοποιηθεί χωρίς να αλλαχτούν (ή από οποιαδήποτε ομάδα ή gid (ταυτότητα ομάδας) καθορισμένο από την GROUP= crtl_interface παράμετρο).

# the default ctrl_interface option used in the example file
	# /usr/share/doc/wpasupplicant/examples/wpa-roam.conf
        ctrl_interface=DIR=/var/run/wpa_supplicant GROUP=netdev

To interact with the supplicant, the wpa_cli (command line) and wpa_gui (QT)
have been provided. With these you may connect, disconnect, add/delete new
network blocks, provide required interactive security information and so on.

Έλεγχος του Roaming Daemon μέσω του wpa_action

Μόλις ξεκινήσει το roaming daemon, αναλαμβάνει τον έλεγχο του ifupdown. Το wpa_cli καλεί το ifup όταν το wpa_supplicant έχει συνεργαστεί επιτυχώς με ένα access point, και καλεί το ifdown όταν η σύνδεση χαθεί ή τερματιστεί. Όσο ο roaming daemon είναι ενεργός, το ifupdown δεν θα πρέπει να ελέγχεται απευθείας με εντολές που δίνονται χειροκίνητα, αλλά δίνετε το /sbin/wpa_action που είναι υπεύθυνο να σταματά και να φορτώνει (reload) τον roaming daemon. Για παράδειγμα, για να σταματήσετε τον romaing daemon στην συσκευή 'eth1':

wpa_action eth1 stop

Όταν απαιτείται να ενημερώσετε τον roaming daemon με λεπτομέρειες για ένα νέο δίκτυο, μπορεί να γίνει χωρίς να τον σταματήσετε. Επεξεργαστείτε το wpa_supplicant.conf αρχείο που χρησιμοποιείται από τον daemon και προσθέστε τις λεπτομέρειες του νέου δικτύου, προσθέστε προαιρετικές ρυθμίσεις δικτύου στο /etc/network/interfaces οι οποίες θα απευθύνονται στο νέο δίκτυο (συνδεδεμένο (linked) μέσω του 'id_str') και μετά κάντε 'reload' (ξανά φορτώστε) τον daemon:

wpa_action eth1 reload

Για τις πλήρεις τεχνικές λεπτομέρειες του τι μπορεί να κάνει το wpa_action ,διαβάστε την wpa_action(8) manpage .

Ρύθμιση λεπτομερειών του Roaming Setup

Μπορεί να έρθετε αντιμέτωποι με καταστάσεις όπου πολλά γνωστά access points (σημεία πρόσβασης) βρίσκονται κοντά. Μπορείτε να επιλέξετε πιο θα προτιμηθεί χειροκίνητα, με το wpa_cli, το wpa_gui ή μπορείτε να δώσετε σε κάθε δίκτυο την δική του προτεραιότητα. Αυτό παρέχεται από την επιλογή 'priority' (προτεραιότητα) του wpa_supplicant.conf .

Το αρχείο ημερολογίου (Logfile)

Όλες οι δραστηριότητες του roaming dameon καταγράφονται στο /var/log/wpa_action.log. Οι παρακάτω πληροφορίες καταγράφονται:

*ώρα και ημερομηνία
*interface name and action event
*values of enviromental variables (μεταβλητές περιβάλλοντος) (WPA_ID, WPA_ID_STR, WPA_CTRL_DIR)
*ifupdown command executed (εκτέλεση εντολής)
*wpa_cli status (βασισμένο στο WPA-PSK, το δίκτυο μπορεί να δίνει διαφορετικές πληροφορίες)
*bssid
*ssid
*id
*id_str
*pairwise_cipher
*group_cipher
*key_mgmt
*wpa_state
*ip_address

Χρήση εξωτερικών script χαρτογράφησης (External Mapping Scripts) (π.χ. guessnet)

Εκτός από την εσωτερική χαρτογράφηση των λογικών διεπαφών (internal mapping of logical interfaces) μέσω του 'id_str', το wpa_action μπορεί να καλέσει εξωτερικά scripts χαρτογράφησης. Ένα script χαρτογράφησης θα πρέπει να επιστρέφει το όνομα της λογικής διεπαφής που πρέπει να καλεστεί. Οποιοδήποτε script χαρτογράφησης που δουλεύει με τον μηχανισμό χαρτογράφησης του ifupdown (δείτε man interfaces) θα πρέπει να δουλέψει και όταν καλείται από το wpa_action.

Για να καλέσετε ένα script χαρτογράφησης, προσθέστε αυτή την γραμμή 'wpa-mapping-script όνομα-του-script' στην interfaces παράγραφο (stanza) της φυσικής συσκευής περιαγωγής (physical roaming device). (Μπορεί να χρειαστεί να δώσετε το απόλυτο μονοπάτι (absolute path) για το script χαρτογράφησης)

Τα περιεχόμενα των γραμμών που ξεκινούν με wpa-map δίνονται στο stdin (συνήθης εισροή) του script χαρτογράφησης. Επειδή το ifupdown επιτρέπει μόνο μια wpa-map γραμμή, μπορείτε να να προσθέσετε οποιοδήποτε αριθμό στο wpa-map για να έχετε επιπλέον γραμμές. Για παράδειγμα:

iface wlan0 inet manual
        wpa-driver wext
        wpa-roam /etc/wpa_supplicant/wpa_supplicant.conf
        wpa-mapping-script guessnet-ifupdown
        wpa-map0 home
        wpa-map1 work
        wpa-map2 school
        # ... additional wpa-mapX lines as required

Εκ προεπιλογής το script χαρτογράφησης θα χρησιμοποιηθεί μόνο όταν δεν υπάρχει 'id_str' για το τρέχων δίκτυο. Εάν θέλετε να απενεργοποιήσετε πλήρως την 'id_str' αντιστοίχηση και να χρησιμοποιείτε μόνο ένα εξωτερικό script χαρτογράφησης, χρησιμοποιήστε την επιλογή 'wpa-mapping-script-priority 1' για να αντικαταστήσετε (override) την συμπεριφορά προεπιλογής.

Εάν το script χαρτογράφησης επιστρέψει ένα άδειο (empty) string, το wpa_action θα χρησιμοποιήσει την διεπαφή προεπιλογής ('default' interface), εκτός και αν μια εναλλακτική είναι καθορισμένη από την 'wpa-roam-default-iface' επιλογή.

Από κάτω είναι ένα προχωρημένο παράδειγμα που χρησιμοποιεί το guessnet-ifupdown σαν εξωτερικό script χαρτογράφησης.

Παράδειγμα /etc/network/interfaces με εξωτερική χαρτογράφηση
/etc/network/interfaces with external mapping example:
allow-hotplug wlan0
iface wlan0 inet manual

        wpa-driver wext
        wpa-roam /etc/wpa_supplicant/wpa_supplicant.conf
        wpa-roam-default-iface default-wparoam
        wpa-mapping-script guessnet-ifupdown
        wpa-map default: default-guessnet
        wpa-map0 home_static
        wpa-map1 work_static


# school can only be chosen via 'id_str' matching
iface school inet dhcp

        # resolvconf
        dns-nameservers 11.22.33.44 55.66.77.88

iface home_static inet static

        address 192.168.0.20
        netmask 255.255.255.0

        network 192.168.0.0
        broadcast 192.168.0.255
        gateway 192.168.0.1
        test peer address 192.168.0.1 mac 00:01:02:03:04:05


iface work_static inet static

        address 192.168.3.200
        netmask 255.255.255.0
        network 192.168.3.0
        broadcast 192.168.3.255

        gateway 192.168.3.1
        test peer address 192.168.3.1 mac 00:01:02:03:04:05

iface default-guessnet inet dhcp

iface default-wparoam inet dhcp

Σε αυτό το παράδειγμα το wpa_action θα χρησιμοποιήσει το guessnet για την επιλογή μιας κατάλληλης λογικής διεπαφής μόνο όταν η 'id_str' επιλογή δεν δόθηκε μέσα στο wpa_supplicant.conf για το τρέχων δίκτυο.

Η 'wpa-map' γραμμές παρέχουν στο guessnet τις λογικές διεπαφές που πρέπει να ελεγχθούν καθώς και την διεπαφή προεπιλογής η οποία θα χρησιμοποιηθεί εάν αποτύχουν όλοι οι έλεγχοι. Οι 'test' γραμμές της κάθε λογικής διεπαφής θα χρησιμοποιηθούν από το guessnet για να διευκρινιστεί το αν είμαστε συνδεδεμένοι σε εκείνο το δίκτυο. Για παράδειγμα, το guessnet θα επιλέξει την λογική διεπαφή 'home_static' εάν υπάρχει μια συσκευή με διεύθυνση IP 192.168.0.1 και MAC 00:01:02:03:04:05 στο τρέχων δίκτυο. Εάν αποτύχουν όλοι οι έλεγχοι, η 'default-guessnet' διεπαφή θα ρυθμιστεί (configured).

Παρακαλώ διαβάστε την guessnet(8) σελίδα εγχειριδίου (manpage) για περισσότερες πληροφορίες.

4. Καταπολέμιση προβλημάτων (troubleshooting)

Για να βρείτε "λάθη" στην σύνδεση ή στην αναγνώριση (authentication), σας συνιστούμε να τρέξετε το `wpa_cli -i <interface>` μέσα σε ένα διαφορετικό κέλυφος (shell) πριν τρέξετε την διεπαφή. Μετά χρησιμοποιήστε την εντολή 'level 0' , για να δείτε όλα τα μηνύματα λαθών (debug messages). Έπειτα χρησιμοποιήστε την `ifup --verbose <interface>` για να δείτε τα μηνύματα λαθών (verbose debug messages) από το script που ξεκινάει το wpasupplicant.

Κρυμμένα ssids

Για πληροφορίες δείτε #358137. Για να μπορέσετε να συνεργαστείτε με κρυμμένα ssids, παρακαλώ δοκιμάστε να βάλετε την επιλογή 'ap_scan=1' στην καθολική ενότητα (global section) και 'scan_ssid=1' στην ενότητα του δικτυακού μπλοκ του wpa_supplicant.conf αρχείου σας. Εάν χρησιμοποιείτε το managed mode, μπορείτε να το κάνετε αυτό με τις πιο κάτω γραμμές:

iface eth1 inet dhcp
wpa-conf managed
wpa-ap-scan 1
wpa-scan-ssid 1
# additional options for your setup

Σύμφωνα με το #368770, η σύνδεση σε δίκτυα με WEP ασφάλεια μπορεί να χρειαστεί πολλή ώρα. Σε ορισμένες περιπτώσεις, ρυθμίζοντας την παράμετρο 'ap_scan=2' στο αρχείο ρυθμίσεων, (ή χρησιμοποιώντας μια 'wpa-ap-scan 2' παράγραφο, το οποίο είναι ισοδύναμο) μπορεί να βοηθήσει πολύ στην ταχύτητα σύνδεση.

Παρακαλώ σημειώστε ότι ρυθμίζοντας το ap_scan στην τιμή 2 απαιτείται από όλα τα δίκτυα να έχουν ακριβώς καθορισμένη πολιτική ασφαλείας (security policy) για τις key_mgmt, pairwise, group και proto μεταβλητές πολιτικής δικτύου (network policy variables).

Ρύθμιση των Δικαιωμάτων Αρχείου

Είναι σημαντικό να παραμένουν τα PSK's και άλλες ευαίσθητες πληροφορίες που αφορούν τις δικτυακές σας ρυθμίσεις ιδιωτικές, γιαυτό σιγουρευτείτε ότι τα σημαντικά αρχεία ρυθμίσεων που περιέχουν τέτοιες λεπτομέρειες μπορούν να διαβαστούν μόνο από τον ιδιοκτήτη τους. Για παράδειγμα:

chmod 0600 /etc/network/interfaces

Εκ προεπιλογής το /etc/network/interfaces μπορεί να διαβαστεί από όλους και γιαυτό είναι ακατάλληλο για να περιέχει μυστικά κλειδιά και passwords.
