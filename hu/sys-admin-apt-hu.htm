Kis APT-Kalauz, receptek

Mit jelent az APT?

Az APT az Advanced Packaging Tool (Fejlett Csomagkezelő Eszköz) rövidítése, ami mögött olyan programok és szkriptek gyűjteménye áll, melyek segítik a rendszergazdát (esetünkben ez a root) a telepítésben és a deb-fájlok kezelésében, de ugyanennyire a magát a rendszert is segíti abban, hogy ismerje a feltelepített csomagokat.

Forráslista

Az "APT" -rendszernek szüksége van egy beállító fájlra, amely a telepíthető és frissíthető csomagok elérési helyéről szolgáltat információt. Általában sources.list-ként szoktunk rá hivatkozni.

Sources are contained in a directory or folder named:
/etc/apt/sources.list.d/

A könyvtárban két fájl található:
/etc/apt/sources.list.d/debian.list és
/etc/apt/sources.list.d/aptosid.list

Ennek előnye az egyszerűbb (automatikus) tükörszerver váltás és a listák biztonságosabb kezelése.

Adhatsz hozzá saját /etc/apt/sources.list.d/*.list fájlokat is

Minden aptosid változat a következő alapértelmezett forrásokat használja:

#aptosid
# Free University Berlin/ spline (Student Project LInux NEtwork), Germany
deb ftp://ftp.spline.de/pub/aptosid/debian/ sid main fix.main
#deb-src ftp://ftp.spline.de/pub/aptosid/debian/ sid main fix.main

További, mindig friss non-free forrásokat is tartalmazó listák a aptosid.list és debian.list címeken találhatók. :

#Debian
# Unstable
deb http://ftp.us.debian.org/debian/ unstable main contrib non-free
#deb-src http://ftp.us.debian.org/debian/ unstable main contrib non-free

# Testing
#deb http://ftp.us.debian.org/debian/ testing main contrib non-free
#deb-src http://ftp.us.debian.org/debian/ testing main contrib non-free

# Experimental
#deb http://ftp.us.debian.org/debian/ experimental main contrib non-free
#deb-src http://ftp.us.debian.org/debian/ experimental main contrib non-free

MEGJEGYZÉS: Példánkban az ftp.us mutatja, hogy USA-ban található tárolókat használtuk. Rootként szerkesztheted ezt a fájlt, beállíthatod, hogy a hozzád legközelebb eső tárolókra hivatkozzon. Egyszerűen változtasd meg az ország jelét oly módon, hogy az ftp.us-ből legyen például ftp.hu, vagy ftp.de. A legtöbb országban található helyi tükörszerver, így jobb sávszélességet kaphatunk, a letöltés is sokkal gyorsabb lehet.

List of Debian Servers and mirrors current status.

A csomagok frissített információinak tárolására az APT egy adatbázist használ. Ez az adatbázis tartalmazza a csomagokat és azt, hogy milyen további csomagok szükségesek egy adott csomag működéséhez (ezt hívjuk függőségnek). Azt apt-get program ezt az adatbázist használja, amikor telepíti az általad kiválasztott csomagokat, megkeresi benne a függőségeket, így biztosítva, hogy választott csomag működjék. Új csomag telepítése

Upgrading packages and installing new packages without stopping X can cause problems. Any method of installing packages under X has this problem.

As long as any package you want to install will not upgrade additional packages then it is safe to install it without stopping X. However if installing the package will cause other packages to upgrade, extreme caution should be taken and unless you are certain the packages which will be upgraded are not currently in use, you should stop X before installing the package.

If you are in any doubt ensure you leave X as per the dist-upgrade instructions before installing any packages. Upgrade an Installed System - dist-upgrade - The Steps .

Csomag törlése

Similarly it should come as no surprise that the following will deinstall a package, however it will not remove the dependencies: 

apt-get remove gaim

Reading package lists... Done
Building dependency tree... Done
The following packages will be REMOVED:
  gaim gaim-irchelper
0 upgraded, 0 newly installed, 2 to remove and 310 not upgraded.
Need to get 0B of archives.
After unpacking 4743kB disk space will be freed.
Do you want to continue [Y/n]?
(Reading database ... 174409 files and directories currently installed.)
Removing gaim-irchelper ...
Removing gaim ...

Ebben az esetben a "gaim" nevű csomag beállító-fájljai nem lettek törölve. Felhasználhatók ha ugyanazt a csomagot újratelepíted, így az eredeti beállításokkal fog ismét működni.

Ha a beállító-fájlokat is törölni akarod, így add ki a parancsot:

apt-get --purge remove gaim

Reading package lists... Done
Building dependency tree... Done
The following packages will be REMOVED:
  gaim*
0 upgraded, 0 newly installed, 1 to remove and 309 not upgraded.
Need to get 0B of archives.
After unpacking 4678kB disk space will be freed.
Do you want to continue [Y/n]? Y
(Reading database ... 174405 files and directories currently installed.)
Removing gaim ...
Purging configuration files for gaim ...

Note the * after the package name in the output of apt. The asterisk in the output of apt indicates that the configuration files will also be removed.

Csomag visszaállítása és/vagy visszatartása

Néha szükséges lehet, hogy egy csomag előző verzióját visszakapjuk, vagy megelőzzük a frissítését a legutóbbi verziójában található súlyos hiba miatt.

Visszatartás (hold):

echo package hold|dpkg --set-selections

A visszatartás levétele:

echo package install|dpkg --set-selections

Visszatartott csomagok listázása:

dpkg --get-selections | grep hold

Visszaállítás

Debian does not support downgrading packages. While it may work in simple cases, it could fail spectacularly for other packages. Please refer to Emergency downgrading.

Despite the fact that it is not supportable, downgrading has been known to work for simple packages. The steps for downgrading a simple package using kmahjongg as an example are:

  1. comment unstable sources in /etc/apt/sources.list.d/debian.list with a #
  2. Add testing sources in /etc/apt/sources.list.d/debian.list , for example:
    deb http://ftp.nl.debian.org/debian/ testing main contrib non-free

  3. apt-get update

  4. apt-get install kmahjongg/testing
  5. Put the newly installed package on hold with:
    echo kmahjongg hold|dpkg --set-selections
  6. comment <#>the testing sources in /etc/apt/sources.list.d/debian.list with and uncomment the unstable sources so that you get back to using unstable sources, then:
  7. apt-get update

Ha később a csomag legújabb verzióját szeretnéd megkapni, így teheted meg:

echo kmahjongg install|dpkg --set-selections
apt-get update
apt-get install kmahjongg

apt-mark

apt-mark is a useful tool that shows you which packages are automatically or manually installed and offers the system administrator the opportunity to mark and hold packages that are not to be removed automatically. Read:

man apt-mark

Telepített rendszer frissítése - dist-upgrade - Overview

Ha a teljes rendszert frissíteni akarod, megteheted a dist-upgrade segítségével. Mindig olvasd el az érvényes figyelmeztetéseket (Current Warnings) a aptosid fő honlapján és ellenőrizd le, hogy a figyelmeztetésben foglaltak érintik-e a te rendszereden lévő, frissítendő csomagokat is. Ha szükséges, tedd visszatartott (hold) státuszba a csomagot, a Csomag visszaállítása és/vagy visszatartása című fejezetben foglaltak szerint.

Egy sima "debian sid upgrade" nem javasolt.

A dist-upgrade gyakorisága

A dist-upgrade alkalmazása teljesen rád van bízva, a heti, kétheti frissítés az ideális, de a biztonság érdekében havonta legalább egy alkalommal ajánlatos. A dist-upgrade 2-3 havonkénti futtatása pedig már a biztonságos időtartam "kicentizése". Azok a csomagok, melyek nem a disztribúció részét képezik, vagy általad, forráskódból kerültek fel, több figyelmet igényelnek a frissítések során, mivel esetleges inkompatibilitásukkal működésképtelenné is tehetik a rendszeredet.

Az adatbázis frissítése után megkeresheted mely csomagokból van új, vagy frissített verzió: (ehhez először az apt-show-versions csomag telepítése szükséges)

apt-show-versions -u
libpam-runtime/unstable upgradeable from 0.79-1 to 0.79-3
passwd/unstable upgradeable from 1:4.0.12-5 to 1:4.0.12-6
teclasat/unstable upgradeable from 0.7m02-1 to 0.7n01-1
libpam-modules/unstable upgradeable from 0.79-1 to 0.79-3.........

Egy csomag és annak függőségei (pl. gcc-4.0) a következők szerint frissíthető:

apt-get install gcc-4.0
Reading package lists... Done
Building dependency tree... Done
gcc-4.0 is already the newest version.
0 upgraded, 0 newly installed, 0 to remove and xxx not upgraded

Csak letöltés

Egy kevésbé ismert, de nagyszerű módszer annak megállapítására, hogy milyen csomagok frissülnek egy dist-upgrade során, a -d kapcsoló használata:

apt-get update && apt-get dist-upgrade -d

A -d lehetővé teszi, hogy a telepítés kihagyásával csak letöltsd a dist-upgrade tartalmát, miközben használod a grafikus felületet, majd egy későbbi időpontban 3-as futási szintre váltva telepítsd azokat. Megadja a lehetőséget, hogy a csomaglistát összehasonlítsd a figyelmeztetésekkel, majd eldönthesd hogy továbblépsz, vagy megszakítod a műveletet, ahogy ezt az alábbi példa illusztrálja:

apt-get dist-upgrade -d
Reading package lists... Done
Building dependency tree

  elinks-data
The following packages have been kept back:
  git-core git-gui git-svn gitk icedove libmpich1.0ldbl

  alsa-base bsdutils ceni configure-ndiswrapper debhelper

Y (yes) válasz esetén letölti a frissített csomagokat, de a rendszeren nem végez változtatásokat.

Miután a "dist-upgrade -d" elkészült a letöltéssel és szeretnéd a rendszerfrissítést befejezni akár azonnal, akár később, kövesd az alábbi instrukciókat:

dist-upgrade - The Steps

SOHA NE végezz dist-upgrade, vagy upgrade műveleteket amíg fut a grafikus felület.
Mindig olvasd el az Aktuális Figyelmezetéseket (Current Warnings) a aptosid fő honlapján. E figyelmeztetések azért vannak, mert a rendszer a Debian fejlesztői ágra (sid) épül, ami naponta változik.
## Log out of KDE.  
## Go to Textmode by doing Ctrl+Alt+F1
## logon as root, and then type:
service kdm stop  ##If using another window manager replace kdm, for example with gdm
apt-get update
apt-get dist-upgrade
apt-get clean
service kdm start ##If using another window manager replace kdm, for example with gdm
SOHA NE FUTTASS DIST-UPGRADE [vagy UPGRADE] műveletet adept, synaptic vagy aptitude használatával.

Ha nem lépsz át 3-as futási szintre (init 3), rendben, a te dolgod... de szóltunk!

Az Okok, hogy dist-upgrade elvégzéséhez miért NE használj mást mint az apt-get

A csomagkezelők, mint az adept, aptitude, synaptic és kpackage nem mindig képesek lekezelni a Sid-ben történő nagy számú és gyakori változásokat (függőségi és névváltozások, karbantartó szkriptek módosulása...).

Ez nem ezen eszközök fejlesztőinek a hibája, ők kiváló programokat készítenek melyek legendásan jók a Debian stabil ágához. Egyszerűen csak nem alkalmasak azokra a nagyon speciális igényekre melyek a Debian Sid-et jellemzik.

Nyugodtan használd őket a csomagok nézegetésére, keresésére, de a tényleges csomagműveleteket (install/remove/dist-upgrade) az apt-get segítségével végezd.

A csomagkezelők, mint az adept, aptitude, synaptic és kpackage legalábbis (a komplex csomagkiválasztás szempontjából) nem-determinisztikusak, keverd ezt egy gyorsan változó célrendszerrel, mint a Sid és talán még rosszabb ha még egy megkérdőjelezhető minőségű külső csomagtároló is akad a képben (mi nem használjuk és nem ajánljuk használatukat, de a felhasználók rendszerein ezek is léteznek) és biztosan katasztrófába torkollik a kaland.

A másik, hogy mindezek a csomagkezelők 5-ös futási szinten működnek és/vagy grafikus felületen, és egy X alatt végzett dist-upgrade (vagy egyszerű "upgrade", ami nem javasolt) helyrehozhatatlan károkat okozhat a rendszeren, lehet, hogy nem ma, vagy holnap, de előbb-utóbb biztosan.

Másrészről az apt-get szigorúan azt csinálja, amire megkéred, bármilyen akadály esetén megállhatsz, megkeresheted, kijavíthatod a hibát. Ha az apt-get el akarja távolítani a fél rendszert (pl. a libek módosulása miatt), akkor az adminisztrátor (ez te vagy) feladata hogy egy komoly felülvizsgálatnak vesse alá a helyzetet.

Ez az oka, amiért a Debian az apt-get használatával épül fel és nem a többi csomagkezelővel.
